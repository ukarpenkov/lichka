# [UI] Последнее сообщение перекрывается блоком ввода при открытой клавиатуре

**Модуль:** `src/pages/chat-room/ChatRoomScreen.tsx`, `src/widgets/message-composer/MessageComposer.tsx`
**Платформа:** Android, iOS
**Приоритет:** P2
**Воспроизводимость:** 100%

## Описание

При активной клавиатуре нижняя часть последнего сообщения в чате скрывается за блоком ввода (`MessageComposer`). Видны только верхние строки текста; нижние углы bubble, время и иконка типа сообщения оказываются под полем ввода.

## Шаги воспроизведения

1. Открыть любой чат с сообщениями.
2. Прокрутить к последнему сообщению (или открыть чат с историей).
3. Тапнуть по полю ввода — появляется клавиатура.
4. Обратить внимание на последнее сообщение над блоком ввода.

## Ожидаемый результат

Последнее сообщение полностью видно над блоком ввода, с небольшим визуальным зазором между bubble и полем «Сообщение…».

## Фактический результат

Нижняя часть bubble последнего сообщения перекрывается блоком ввода. Timestamp и иконка уведомления прижаты к верхней границе composer или скрыты.

## Причина

`MessageComposer` при открытой клавиатуре поднимается визуально через `translateY: -KEYBOARD_COMPOSER_GAP` (16 px), но остаётся в том же flex-слоте под `FlatList`. Список сообщений не получает дополнительный `paddingBottom`, поэтому контент в нижней части viewport перекрывается смещённым composer.

## Исправление

В `ChatRoomScreen` добавить анимированный `paddingBottom` для `contentContainerStyle` списка сообщений при `keyboardHeight > 0`. Константа `CHAT_LIST_KEYBOARD_BOTTOM_INSET` (`KEYBOARD_COMPOSER_GAP + 12`) вынесена в `shared/lib/keyboard.ts`.
