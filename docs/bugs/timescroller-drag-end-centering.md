# TimeScroller — отсутствие центрирования при медленной прокрутке

**Дата:** 2026-07-05

## Описание

После завершения прокрутки списка часов или минут выбранное значение не всегда выравнивается по центральной линии (highlight). Проблема возникает при медленной прокрутке или когда пользователь отпускает палец без заметной инерции.

## Причина

Предыдущий фикс (`timescroller-scroll-inertia-centering.md`) удалил `onScrollEndDrag`, чтобы не ломать инерцию при быстром свайпе. Однако при нулевой или минимальной скорости (`velocity.y ≈ 0`) инерция не возникает и `onMomentumScrollEnd` не вызывается. Список остаётся в промежуточной позиции, не доведённый до центра.

## Воспроизведение

1. Открыть пикер времени.
2. Медленно прокрутить список часов или минут.
3. Отпустить палец без резкого движения.
4. Значение остаётся смещённым относительно центрального highlight.

## Исправление

Добавлены обработчики `onScrollEndDrag` для обоих FlatList (часы и минуты). Обработчик проверяет скорость прокрутки (`velocity.y`): если она меньше порога `0.01`, вызывается та же логика центрирования, что и в `onMomentumScrollEnd`. При наличии инерции (скорость выше порога) обработчик ничего не делает — финальный снап произойдёт через `onMomentumScrollEnd`.
