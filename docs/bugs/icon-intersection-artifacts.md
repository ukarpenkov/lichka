# [UI] Визуальный дефект иконок: яркие точки (артефакты) в местах пересечения линий

**Дата:** 2026-07-02
**Приоритет:** Low
**Статус:** open

## Описание

При отрисовке векторных иконок (lucide-react-native + react-native-svg) наблюдается дефект рендеринга контуров: в точках пересечения линий образуются локальные утолщения / яркие пятна.

## Корневая причина

Иконки Lucide рендерятся как набор отдельных SVG-элементов (`<path>`, `<rect>`, `<circle>`) с `fill="none"` и `strokeWidth=2`. Когда два элемента пересекаются, их штрихи накладываются друг на друга, создавая вдвое большую плотность пикселей (double-stroke overlap). Это особенно заметно:

- На Android (из-за особенностей Canvas-рендеринга)
- При малом размере иконки (24px), где 1px перекрытия составляет существенную долю

## Затронутые иконки

### 1. CalendarDays (нижняя панель)
- `<rect>` x:3, y:4, w:18, h:18 + `<path>` d="M3 10h18"
- Горизонтальная линия пересекает rect, создавая double-stroke на левой и правой гранях

### 2. Settings / Gear (нижняя панель)  
- Сложный путь шестерни + отдельный `<circle>` в центре
- Возможно наложение штрихов на внутреннем кольце

### 3. MessageCircle (нижняя панель)
- Единый `<path>`, точка соединения хвостика с кругом обрабатывается strokeLinejoin
- При остром угле соединения anti-aliasing создаёт видимый артефакт

### 4. AlarmClock (панель действий чата)
- `<circle>` + ножки будильника, пересекающие границу круга
- Double-stroke в местах крепления ножек

### 5. Mic (панель действий чата)
- `<rect>` корпуса + `<path>` дуги, пересекающей верхнюю часть корпуса
- Double-stroke на пересечении дуги и корпуса

## Воспроизведение

- Платформа: Android / iOS
- Размер иконок: 20-24px
- Цвет: любой (дефект проявляется на всех цветах)

## Решение

Для иконок с пересекающимися элементами использовать `<ClipPath>` в `react-native-svg`, чтобы обрезать штрих пересекающего элемента до внутренней границы базового элемента. Это предотвращает double-stroke overlap на стыках.
