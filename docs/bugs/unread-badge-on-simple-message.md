# Бейдж непрочитанных после обычного сообщения

**Статус:** fixed  
**Дата:** 2026-07-22

## Симптом

После входа в чат, отправки обычного (simple) сообщения и возврата к списку чатов появляется бейдж непрочитанных.

## Ожидание

Бейдж только для сработавших уведомлений приложения: отложенное напоминание (`reminder`), будильник (`alarm`), периодическое (`periodic`).

## Причина

`getUnreadCounts()` считал **все** сообщения с `created_at` позже `last_read_at`.  
`markChatAsRead` вызывался только при фокусе чата, поэтому сообщение, написанное после входа, оставалось «непрочитанным».

## Исправление

- Счётчик ограничен типами `reminder` / `alarm` (уже сработавшие) и `periodic` (есть fire после last read).
- При уходе из чата (`useFocusEffect` cleanup) тоже вызывается `markChatAsRead`.
