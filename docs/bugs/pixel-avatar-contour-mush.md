# Баг: пиксельный контур аватара превращается в «кашу»

**Ветка:** `feat/pixel-avatar`  
**Статус:** open  
**Платформа:** Android (воспроизведено), ожидаемо и iOS  
**Дата:** 2026-07-22

## Описание проблемы

После выбора фото для аватара чата приложение под капотом прогоняет его через локальный пайплайн `features/pixel-avatar` (JPEG/PNG decode → crop → Sobel → pixel grid → PNG).

**Ожидание:** сильно упрощённая пиксельная версия **только с контурами** силуэта и лица — в духе референса GameBoy / 8-bit (чёрные линии на белом, читаемое лицо), как в `docs/design/refs/pixel-contour-avatar-example.png`.

**Факт:** на реальных фото (например портрет ребёнка на кухне) получается **неузнаваемая «каша»**:
- либо плотная цветная/блочная мозаика вместо тонких контуров;
- либо редкий чёрно-белый шум / обрывки линий без читаемого лица и силуэта.

То есть цепочка преобразований **отрабатывает без падения**, но **визуальный результат не соответствует цели фичи**.

## Шаги воспроизведения

1. Собрать приложение с ветки `feat/pixel-avatar`.
2. Создать / редактировать чат → «Фото».
3. Выбрать из галереи обычное JPEG-фото (не студийный портрет на размытом фоне): лицо сбоку, сложный фон (шкафы, текстуры).
4. Смотреть превью аватара в `ChatForm` и сохранённый `media/avatars/{id}.png`.

## Ожидаемый результат

Пиксельный контурный аватар: крупные чёткие блоки линий, белый фон, узнаваемый силуэт/черты лица (как на референсе).

## Фактический результат

Нечитаемый pixel-noise / «каша»; контур лица и силуэта не считываются.

## Что уже пробовали в этой ветке (и почему всё ещё баг)

1. Цветные контуры + низкий порог Sobel → цветная заливка фото («каша»).
2. NMS / высокий порог / density grid-snap → либо шум, либо пусто.
3. Mono + белый фон + Sobel на низкой сетке + skin-crop + suppress border-runs → на сложных кадрах всё ещё неузнаваемо.

Вывод: текущий **чисто эвристический** пайплайн без надёжного face/subject crop **не даёт стабильного контурного аватара** на реальных галерейных фото.

## Гипотезы корневой причины

1. **Нет детекции лица / субъекта** — на кадрах «лицо сбоку + кухня» Sobel цепляет шкафы и текстуры сильнее лица; skin-gate слишком грубый.
2. **Потеря семантики при сильном даунскейле** — сетка ~48–56 px без предварительного face-ROI не удерживает глаза/нос/рот.
3. **Один набор defaults на все фото** — studio portrait vs cluttered snapshot требуют разных порогов; адаптивности нет.
4. Возможный вклад платформы: на Android раньше были сбои decode (`expected JPEG`, `pako.deflate`) — их чинили отдельно; текущий баг именно **про качество контура**, не про крэш.

## Предлагаемое направление фикса

1. **Face / subject ROI** перед пикселизацией (Vision / ML Kit или аналог) — кадрировать лицо, затем контуры.
2. Подогнать алгоритм под эталон на наборе референсных фото (студия + cluttered) с метрикой «читаемость силуэта».
3. Пока ROI нет — не выдавать результат как «готовый avatar style»; либо превью + ручной crop, либо fallback на обычное фото / иконку.
