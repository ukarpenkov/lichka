# Баг: пиксельный контур аватара превращается в «кашу»

**Ветка:** `feat/pixel-avatar`  
**Статус:** resolved — внедрён **theme-pixel** (won't-fix contour MVP)  
**Платформа:** Android (воспроизведено), ожидаемо и iOS  
**Дата:** 2026-07-22  
**Обновлено:** 2026-07-22  
**Закрыто:** 2026-07-22 — см. `docs/features/theme-pixel-avatar-proposal.md`

## Описание проблемы

После выбора фото для аватара чата приложение под капотом прогоняет его через локальный пайплайн `features/pixel-avatar` (JPEG/PNG decode → crop → Sobel → pixel grid → PNG).

**Ожидание:** сильно упрощённая пиксельная версия **только с контурами** силуэта и лица — в духе референса GameBoy / 8-bit (чёрные линии на белом, читаемое лицо), как в `docs/design/refs/pixel-contour-avatar-example.png`.

**Факт:** на реальных фото (например портрет ребёнка на кухне) получается **неузнаваемая «каша»**:
- либо плотная цветная/блочная мозаика вместо тонких контуров;
- либо редкий чёрно-белый шум / обрывки линий без читаемого лица и силуэта.

То есть цепочка преобразований **отрабатывает без падения**, но **визуальный результат не соответствует цели фичи**.

## Шаги воспроизведения

1. Собрать приложение с ветки `feat/pixel-avatar`.
2. Создать / редактировать чат → «Фото».
3. Выбрать из галереи обычное JPEG-фото (не студийный портрет на размытом фоне): лицо сбоку, сложный фон (шкафы, текстуры).
4. Смотреть превью аватара в `ChatForm` и сохранённый `media/avatars/{id}.png`.

## Ожидаемый результат

Пиксельный контурный аватар: крупные чёткие блоки линий, белый фон, узнаваемый силуэт/черты лица (как на референсе).

## Фактический результат

Нечитаемый pixel-noise / «каша»; контур лица и силуэта не считываются.

## Что уже пробовали в этой ветке (и почему всё ещё баг)

1. Цветные контуры + низкий порог Sobel → цветная заливка фото («каша»).
2. NMS / высокий порог / density grid-snap → либо шум, либо пусто.
3. Mono + белый фон + Sobel на низкой сетке + skin-crop + suppress border-runs → на сложных кадрах всё ещё неузнаваемо.

Вывод: текущий **чисто эвристический** пайплайн без надёжного face/subject crop **не даёт стабильного контурного аватара** на реальных галерейных фото.

## Гипотезы корневой причины

1. **Нет детекции лица / субъекта** — на кадрах «лицо сбоку + кухня» Sobel цепляет шкафы и текстуры сильнее лица; skin-gate слишком грубый.
2. **Потеря семантики при сильном даунскейле** — сетка ~48–56 px без предварительного face-ROI не удерживает глаза/нос/рот.
3. **Один набор defaults на все фото** — studio portrait vs cluttered snapshot требуют разных порогов; адаптивности нет.
4. Возможный вклад платформы: на Android раньше были сбои decode (`expected JPEG`, `pako.deflate`) — их чинили отдельно; текущий баг именно **про качество контура**, не про крэш.

## Предлагаемое направление фикса (устарело как основной путь)

Ранее рассматривалось:
1. Face / subject ROI перед контурами (Vision / ML Kit).
2. Тонкая настройка Sobel под эталон.
3. Fallback на обычное фото / иконку.

Контур «как на референсе» с произвольного gallery-фото **без ROI/ML текущим пайплайном нереалистичен**. Основной выход — см. ниже.

## Выход из ситуации: theme-pixel avatar (вместо контуров)

**Решение:** не гнаться за edge-only контуром. Делать **лёгкую пиксельность** + «ЧБ», но палитра **только из цветов выбранной темы** (`background` / `text` и 2–4 промежуточных lerp/alpha между ними).

Это совпадает с языком Lichka: темы уже двухцветные, Streamline Pixel иконки чатов тоже тинтятся одним `text`.

### Алгоритм (целевой)

1. Center crop в квадрат (face-detect опционален later, не блокер).
2. Умеренный даунскейл до сетки **~32–48** (box / nearest) — «чуть пиксельно», не экстремальный 16×16.
3. Luminance пикселя → 2–4 уровня яркости (posterize).
4. Уровни красить палитрой темы, например:
   - самый светлый → `background`
   - самый тёмный → `text`
   - середины → смеси `background`↔`text` (`withAlpha` / lerp)
5. Nearest-neighbor upscale до размера аватара; сохранить PNG **или** хранить luminance-маску и перекрашивать при смене темы на лету.

### Почему это закрывает баг

| Контуры (сейчас) | Theme-pixel |
|------------------|-------------|
| Sobel ломается на кухне / лице сбоку | Форма лица читается пятнами яркости |
| Нужен face ROI для эталона | ROI не обязателен для приемлемого вида |
| Не бьётся с 2-color UI | Нативно под 13 тем приложения |
| Референс-контур недостижим эвристиками | Реалистичный scope для `feat/pixel-avatar` |

### Product-решение

- Снять цель «как `pixel-contour-avatar-example.png`» с MVP.
- Контурный режим — optional later (только с face-detect).
- Переименовать/переписать proposal под **theme-pixel chat avatar**; заменить `processPixelContourBuffer` на luminance→theme quantize.
- Статус этого бага после внедрения theme-pixel: **resolved** (won't-fix contour MVP) либо закрыть с ссылкой на новую фичу.

### Внедрено (2026-07-22)

- `processThemePixelBuffer` / `createThemePixelAvatar` — luminance → theme palette.
- `ChatForm` передаёт `background` + `text` из `useTheme`.
- Proposal: `docs/features/theme-pixel-avatar-proposal.md`.
- Контурный proposal помечен `rejected` / superseded.
- Тюнинг: сетка ~96, 16 оттенков, duotone-ramp (на dark-теме светлое → text/зелёный, не инверсия).
