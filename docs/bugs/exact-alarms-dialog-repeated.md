# Exact alarms guide показывается при каждом создании будильника

**Дата:** 2026-07-03
**Приоритет:** Medium
**Статус:** fixed

## Описание

Информационное сообщение с инструкциями по настройке разрешений (Exact alarms, Battery optimization, Show over lock screen, Auto-start) отображается **каждый раз** при создании нового будильника.

## Ожидаемое поведение

Сообщение должно показываться **только один раз**. После того как пользователь его увидел (закрыл или нажал «Готово»), оно больше не должно появляться.

## Фактическое поведение

Сообщение появляется при каждой установке будильника. Пользователь вынужден закрывать одно и то же сообщение при каждом действии.

## Шаги воспроизведения

1. Открыть приложение
2. Создать первый будильник → сообщение показывается
3. Создать второй будильник → сообщение показывается снова
4. Создать третий будильник → сообщение снова

## Причина

В `src/widgets/message-composer/MessageComposer.tsx:141-143` `handlePickerConfirm` безусловно вызывает `setAlarmGuideVisible(true)` при создании будильника, без проверки флага о том, был ли диалог уже показан.

## Решение

Сохранять флаг `alarm_guide_shown` в AsyncStorage при показе диалога. При последующих созданиях будильника проверять флаг и пропускать показ гайда, если он уже был показан.
