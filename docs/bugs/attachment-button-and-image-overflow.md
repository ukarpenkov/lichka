# Баги интерфейса чата: кнопка вложений и переполнение изображений

**Дата:** 2026-07-04
**Приоритет:** 🔴 Bug #2 — критичный, 🟡 Bug #1 — средний
**Затрагивает:** `widgets/message-composer`, `widgets/image-message`, `pages/chat-room/MessageBubble`

---

## Bug #1: Кнопка вложений вне поля ввода

### Фактический результат
Кнопка «скрепка» (Paperclip) находится в панели инструментов (`actions`) под полем ввода текста, слева от кнопок микрофона, повтора, будильника и т.д.

### Ожидаемый результат
Кнопка «скрепка» должна быть внутри поля ввода текста, справа (справа от placeholder «Сообщение…»), аналогично тому, как это реализовано в Telegram/WhatsApp.

### Техническая причина
`IconButton(Paperclip)` находится в строке `actions` (строка 365 `MessageComposer.tsx`), которая рендерится **под** `TextInput`. Кнопка должна быть размещена внутри контейнера поля ввода в `flexDirection: 'row'`.

### Влияние
- Нарушение UX-паттерна, привычного пользователям мессенджеров
- Визуально кнопка «оторвана» от поля ввода

## Bug #2: Изображение выходит за границы bubble сообщения

### Фактический результат
Прикреплённое изображение не вписано в bubble сообщения — выходит за его границы по горизонтали, нарушая визуальную целостность чата.

### Ожидаемый результат
Изображение должно быть полностью contained внутри bubble сообщения с правильным border radius (16px, как у bubble) и отступами. Изображение должно заполнять bubble от края до края, а bubble скруглять углы через `overflow: 'hidden'`.

### Техническая причина
В `ImageMessage.tsx` (строка 49):
```ts
const bubbleWidth = screenWidth * 0.8;
```
Ширина изображения равна `screenWidth * 0.8`, что совпадает с `maxWidth: '80%'` bubble в `MessageBubble.tsx`. Однако bubble имеет `paddingHorizontal: 12`, и итоговая ширина содержимого bubble = `screenWidth * 0.8 + 24`, что превышает `maxWidth`. Изображение **переполняет** bubble на 24px (по 12px с каждой стороны).

### Влияние
- Критичное визуальное нарушение: изображение «вылезает» за границы сообщения
- Нарушение целостности дизайна чата

---

## Решение

### Bug #1
Обернуть `TextInput` и `IconButton(Paperclip)` в общий контейнер со стилем поля ввода (`border`, `borderRadius: 20`, `flexDirection: 'row'`). Убрать Paperclip из строки `actions`.

### Bug #2
Использовать отрицательные отступы (`marginHorizontal: -12`, `marginVertical: -8`) в `ImageMessage` для компенсации padding bubble. Изображение будет рендериться от края до края bubble. Добавить `overflow: 'hidden'` в bubble для изображений, чтобы `borderRadius: 16` обрезал углы изображения.
