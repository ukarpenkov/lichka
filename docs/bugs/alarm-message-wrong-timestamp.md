# 🐛 Баг: Неверная метка времени у сообщения будильника

## Описание
Когда срабатывает будильник, сообщение появляется в чате с временем **создания** будильника, а не с временем **его срабатывания**.

## Шаги воспроизведения
1. Открыть чат, текущее время — `12:00`.
2. Нажать кнопку отправки → выбрать тип сообщения **«Будильник»**.
3. Установить время срабатывания — `12:05`.
4. Подтвердить создание.
5. Дождаться срабатывания будильника в `12:05`.

## Ожидаемое поведение
В чате появляется сообщение будильника с меткой времени `12:05` (момент срабатывания).

## Фактическое поведение
В чате появляется сообщение будильника с меткой времени `12:00` (момент создания).

## Пример
```
Сейчас: 12:00
Создан будильник на: 12:05
Срабатывание: 12:05

Отображается в чате: 12:00  ❌
Должно отображаться:   12:05  ✅
```

## Вероятная причина
При сохранении сообщения в SQLite в поле `createdAt` / `timestamp` записывается `Date.now()` в момент **создания** будильника (в UI-слое), а не в момент **его срабатывания** (в сервисе/воркере).

Скорее всего проблема в одном из мест:
- `createMessage()` вызывается сразу при создании будильника, а не в обработчике срабатывания.
- В планировщике (alarm worker / notification handler) используется сохранённый `scheduledAt` вместо `Date.now()` при фактической отправке сообщения в чат.
- При «пробуждении» сообщения из очереди `triggeredAt` не перетирает `createdAt`.

## Предложение по фиксу
1. Разделить два поля в модели сообщения:
   - `scheduledAt` — когда запланировано (для будильников/напоминаний).
   - `createdAt` — когда сообщение фактически появилось в чате.
2. В обработчике срабатывания будильника записывать `createdAt = Date.now()` на момент срабатывания.
3. В UI отображать `createdAt` как метку времени в «пузыре» сообщения.
4. `scheduledAt` оставить для сортировки в табе **Scheduled** и для иконок/бейджей «запланировано».

## Тест-кейс (добавить в `*.test.ts`)
```ts
it('alarm message createdAt equals trigger time, not creation time', async () => {
  const createdAt = Date.now();            // 12:00
  const triggerAt = createdAt + 5 * 60_000; // 12:05

  jest.setSystemTime(createdAt);
  await scheduleAlarm({ triggerAt, text: 'Wake up' });

  jest.setSystemTime(triggerAt);
  await flushScheduledQueue();

  const messages = await getMessages();
  expect(messages[0].createdAt).toBe(triggerAt);
  expect(messages[0].scheduledAt).toBe(triggerAt);
});
```

## Приоритет
🟡 **Medium** — влияет на UX и доверие к таймлайну чата, но не ломает функционал.

## Затронутые модули (предварительно)
- `src/features/messages/` — модель сообщения
- `src/features/alarms/` (или `scheduled/`) — воркер срабатывания
- `src/features/chats/` — рендер метки времени в «пузыре»
