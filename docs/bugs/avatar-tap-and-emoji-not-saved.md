# Баг: Тап по аватарке + emoji не сохраняется в БД

## Описание
Два связанных бага:
1. Аватар-превью в форме чата не было кликабельно — тап по нему ничего не делал. Нужно: нет аватара → открыть галерею, есть emoji → открыть emoji-пикер.
2. При сохранении чата с emoji-аватаром, emoji не записывалось в базу данных — `avatarPath` становился `null`.

## Причина
1. Аватар-превью рендерилось как статичный `View`/`Image` без `onPress`.
2. В `handleSave` при `emojiAvatar` стояло `avatarPath = null` вместо `avatarPath = emojiAvatar`.

## Решение
1. Обёрнул avatar-превью в `Pressable` с `handleAvatarTap`: проверяет наличие `emojiAvatar` и открывает соответствующий пикер.
2. Исправлено на `avatarPath = emojiAvatar`.
3. При редактировании чата корректно определяется тип аватара (emoji vs фото) по наличию `/` или `\` в пути.
