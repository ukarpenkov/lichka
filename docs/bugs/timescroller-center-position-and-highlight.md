# TimeScroller — смещение минут из центра и подсветка предыдущего часа

## Описание

В компоненте `TimeScroller` (двухколоночный вертикальный пикер часов/минут с центральной
голубой областью выбора) выбранное время отображается некорректно:

- **Минуты** смещены относительно центра и не попадают в голубую область выбора.
- **Часы**: голубым цветом подсвечивается предыдущее значение вместо текущего выбранного.

Ожидаемое поведение: предыдущий час/минута — сверху, выбранные (текущие) — строго по центру
в голубой области выбора, следующие — снизу. Подсветка применяется только к выбранному значению.

## Причина

Накопились три взаимоисключающие off-by-one поправки, внесённые разными коммитами:

1. `ee328c2` добавил `-1` в `dataIdxForHour`/`dataIdxForMinute` и `+1` в `normalizeHour`/
   `normalizeMinute`. На момент коммита `scrollToDataIndex` использовал `offset = dataIdx * ITEM_HEIGHT`
   (без `-1`), поэтому комбинация была корректной: элемент `24 + h` попадал в центр.
2. `f888ab7` изменил `scrollToDataIndex` на `offset = (dataIdx - 1) * ITEM_HEIGHT` (и так же в
   `recenter*`), **не тронув** `dataIdxForHour`. Появился **двойной** `-1` → целевой элемент стал
   смещаться на одну позицию вверх, в голубую область попадал `h - 1` вместо `h`.
3. Незакоммиченные (staged) изменения ввели индексную подсветку через состояние `centerHourIdx`/
   `centerMinIdx` и сравнение `index === centerIdx` в `renderItem`. При этом `centerHourIdx`
   устанавливался в `idx = round(offset / ITEM_HEIGHT)` — это индекс **верхней** видимой строки,
   а не центральной. Голубой цвет получал элемент выше области выбора.

Итог: голубая область содержала `h - 1`, а голубой цвет — элемент на верхней строке (`h - 2`
относительно выбранного), плюс `normalizeHour(idx)` сообщал значение центра как `(idx + 1) % N`,
что не совпадало с фактической центральной позицией после двойного смещения.

## Воспроизведение

1. Открыть `DateTimePicker` (экран планирования/будильника).
2. Задать любое время, например `05:30`.
3. В колонке часов голубым подсвечено `04` (или значение над центром), а не `05`.
4. В колонке минут центральная голубая ячейка не совпадает с `30` — значение смещено.

## Исправление

Приведение к единой непротиворечивой модели:

- `highlight` расположен на `top: ITEM_HEIGHT * 1` → выбранный элемент в центральной строке
  (координата `y = ITEM_HEIGHT`).
- `scrollToDataIndex` использует `offset = (dataIdx - 1) * ITEM_HEIGHT` → элемент `dataIdx`
  попадает в центр. Следовательно `dataIdxForHour(h)` должен возвращать **индекс значения `h`
  в средней копии** = `MIDDLE_COPY * HOUR_COUNT + h` (без `-1`).
- При скролле `idx = round(offset / ITEM_HEIGHT)` — индекс **верхней** строки; центральная =
  `idx + 1`. Следовательно `centerHourIdx = idx + 1` (не `idx`).
- `normalizeHour(idx) = (idx + 1) % N` остаётся корректным (центральная = верх + 1) — не менялся.

Изменения (6 точечных правок):

1. `dataIdxForHour` → `MIDDLE_COPY * HOUR_COUNT + h` (убран `-1`).
2. `dataIdxForMinute` → `MIDDLE_COPY * MIN_COUNT + m` (убран `-1`).
3. `handleHourScroll` → `setCenterHourIdx(idx + 1)`.
4. `handleHourScrollEnd` → `setCenterHourIdx(idx + 1)`.
5. `handleMinuteScroll` → `setCenterMinIdx(idx + 1)`.
6. `handleMinuteScrollEnd` → `setCenterMinIdx(idx + 1)`.

`handleHourPress`/`handleMinutePress` уже используют `setCenterHourIdx(dataIdxForHour(real))`,
что после исправления `dataIdxForHour` равно индексу центрального элемента — остаётся без изменений.
