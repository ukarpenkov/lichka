# ImageViewer — белый блинк и проблемы с жестами

**Дата:** 2026-07-06

## Описание

В компоненте `ImageViewer` (`src/features/image-viewer/`) выявлены 4 связанных дефекта.

### 1. Белый блинк при открытии

При тапе на превью изображения в чате — перед показом картинки экран на ~100–200мс становится полностью белым, затем появляется изображение.

### 2. Невозможность панорамирования после зума

После увеличения (pinch или double-tap) жест Pan не работает — нельзя сдвинуть увеличенное изображение для просмотра другой области.

### 3. Белый блинк при закрытии

При нажатии на крестик (X) или swipe-down — перед возвратом в чат белая вспышка.

### 4. Первое нажатие после закрытия не работает

После закрытия ImageViewer первое нажатие на то же превью в чате не открывает viewer. Второе нажатие — работает.

## Шаги воспроизведения

1. Открыть чат с изображением, тапнуть по превью → **белая вспышка** перед картинкой (#1).
2. Pinch-to-zoom → попытаться сдвинуть картинку одним пальцем → **не двигается** (#2).
3. Нажать X → **белая вспышка** перед возвратом в чат (#3).
4. Сразу тапнуть по превью → **не открывается**; тапнуть ещё раз → открывается (#4).

## Ожидаемый результат

- Плавный fade-in/fade-out без белых вспышек.
- Pan работает после зума.
- Первое нажатие на превью после закрытия открывает viewer.

## Фактический результат

Описано выше.

## Окружение

- **Компонент:** `src/features/image-viewer/ImageViewer.tsx`, `useImageViewer.ts`
- **RN:** 0.85.3
- **Жесты:** `react-native-gesture-handler`, `react-native-reanimated`

## Причина

### #1 и #3 — белый фон Modal

`<Modal transparent={false}>` имеет дефолтный белый фон. Поверх него `overlay` с `backgroundColor: background` анимирует `opacity` 0→1 (открытие) и 1→0 (закрытие). Пока `opacity` близок к 0, сквозь прозрачный overlay виден **белый фон модала**, а не чат позади.

### #2 — `Gesture.Exclusive(pinch, pan)`

`Gesture.Exclusive` заставляет pinch и pan конкурировать. Когда pinch побеждает (2 пальца), pan переходит в состояние `FAILED` и **не реактивируется**, когда пользователь отпускает один палец и пытается панорамировать оставшимся. Жест pan остаётся "проигравшим" до конца сессии касания.

### #4 — отложенный колбэк закрытия

`close()` вызывает `overlayOpacity.value = withTiming(0, {duration: 200}, () => runOnJS(onClose)())`. `onClose` (→ `setVisible(false)`) срабатывает **через 200мс**, а не сразу. Если пользователь тапнет превью в это окно:

1. `open()` ставит `visible=true` (но он уже `true` — onClose ещё не отработал) → состояние не меняется.
2. Через 200мс pending-колбэк вызывает `onClose` → `visible=false` → модал закрывается.
3. Тап пользователя «съеден» — следующий тап работает, потому что `visible` уже `false` и `open()` реально меняет состояние.

Дополнительно: во время fade-out полупрозрачный overlay модала перехватывает тапы по превью позади.

## Решение

- **#1/#3:** `transparent` Modal + фейд overlay поверх чата. Белый фон модала больше не виден.
- **#2:** `Gesture.Simultaneous(pinch, pan)` вместо `Exclusive` — pan остаётся активным во время pinch и берёт управление, когда pinch завершается. Флаг `isPinching` блокирует pan во время активного pinch.
- **#4:** Локальный `internalVisible` декаплинг mount/unmount от родительского `visible`. `onClose` вызывается синхронно, fade-out управляется внутренним состоянием. `cancelAnimation` отменяет pending-колбэк при reopen. `pointerEvents="none"` на overlay во время закрытия — тапы проходят к превью позади.
- **Бонус:** fade-in изображения через `onLoad` (`imageOpacity`) — нет пустого фрейма на медленной загрузке.
