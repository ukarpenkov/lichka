# [UX] Клавиатура не открывается автоматически при активации поиска в списке чатов

**Приоритет:** Medium

## Описание
При нажатии на иконку поиска в экране списка чатов поле ввода появляется, но программная клавиатура не открывается автоматически. Требуется дополнительный тап по полю ввода.

## Локализация
`src/pages/chat-list/GlobalSearch.tsx`

## Причина
В `GlobalSearch.tsx` вызов `inputRef.current?.focus()` был обёрнут в `InteractionManager.runAfterInteractions`, что откладывало фокус до завершения всех анимаций. В результате клавиатура не успевала появиться синхронно с монтированием компонента.

## Решение
1. Удалён `InteractionManager` из импортов.
2. Удалён `useEffect` с `InteractionManager.runAfterInteractions`.
3. На `<TextInput>` добавлен проп `autoFocus`, который гарантирует мгновенный фокус и вызов клавиатуры при монтировании компонента.
4. Оставлен `useEffect` для сброса `query` и `results` при скрытии поиска.
