# Баг: Будильник не открывает полноэкранный алерт при активном экране

**Дата:** 2026-07-05
**Severity:** Critical (P0)
**Платформа:** Android

## Описание

При срабатывании будильника с включённым экраном и свёрнутым приложением отображается обычное push-уведомление вместо полноэкранного `AlarmActivity`. Полноэкранный алерт открывается только после тапа по уведомлению.

## Корневая причина

`AlarmReceiver.kt:39-45` полагается исключительно на `Notification + setFullScreenIntent()` для запуска `AlarmActivity`. Однако на Android 10+ `setFullScreenIntent` автоматически запускает Activity **только когда экран заблокирован**. При разблокированном экране система показывает heads-up-уведомление вместо полноэкранного интента.

## Шаги воспроизведения

1. Создать сообщение типа «будильник» с временем +1–2 мин
2. Свернуть приложение (Home), экран **не выключать**
3. Дождаться срабатывания
4. Факт: появляется уведомление в шторке, алерт не открывается
5. Тапнуть по уведомлению → алерт открывается

## Решение

В `AlarmReceiver.handleAlarm()` для случая `isAlarm == true` **напрямую стартовать `AlarmActivity`**, а не только постить уведомление. Приложение имеет `USE_EXACT_ALARM` в манифесте — это даёт право на запуск Activity из BroadcastReceiver, сработавшего от `AlarmManager.setAlarmClock()`.

Уведомление с `fullScreenIntent` остаётся как fallback на случай, если `startActivity` не сработает.

## Изменённые файлы

- `android/.../AlarmReceiver.kt` — добавлен `context.startActivity(alarmIntent)`
- `android/.../AlarmActivity.kt` — сброс уведомления при старте Activity
