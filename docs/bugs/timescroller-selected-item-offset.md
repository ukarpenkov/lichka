# TimeScroller — смещение индикации выбранного элемента на -1

## Описание
В компоненте `TimeScroller` голубая подсветка (accent color) применяется к элементу **выше** центрального, а не к элементу в активной зоне (тёмная ячейка по центру). Визуально пользователь видит выбранным верхнее значение, хотя фактически передаётся центральное.

## Причина
В `scrollToOffset` использовалось смещение `(index + PAD) * ITEM_HEIGHT`, что помещало целевой элемент на **верхнюю** позицию видимой области (позиция 0 в списке из 3 элементов). Активная ячейка (highlight) при этом находится по центру — на позиции `ITEM_HEIGHT * 1`. В результате акцентный цвет получал элемент на позиции 0, а элемент в центре (фактически выбранный) отображался нейтральным цветом.

## Воспроизведение
1. Открыть пикер времени (экран «Показания счётчиков»).
2. Обратить внимание на соотношение голубого цвета и позиции элемента в тёмной центральной ячейке.

## Исправление
Убрана поправка `PAD` из вычислений `scrollToOffset` и обработчиков скролла. Теперь `scrollToOffset` скроллит на `index * ITEM_HEIGHT`, а `idx` вычисляется как `Math.round(contentOffset.y / ITEM_HEIGHT)` без вычитания `PAD`. Это помещает выбранный элемент точно в центр (в highlight-ячейку).
