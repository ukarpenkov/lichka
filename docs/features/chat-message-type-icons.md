# Иконки типа сообщения в чате

**Статус:** implemented

## Описание проблемы

Сейчас в чате все сообщения выглядят одинаково, за исключением голосовых. Нет визуального различия между типами сообщений (напоминание, будильник, периодическое). Пользователь не может быстро понять, что сообщение имеет особый тип, не открывая контекстное меню или не заходя в экран запланированных сообщений.

## Предлагаемое решение

Добавить маленькую иконку типа сообщения рядом с временем отправки в баббле сообщения. Иконка повторяет те же иконки, что используются в блоке отправки сообщений (MessageComposer):

| Тип | Иконка | Источник |
|-----|--------|----------|
| `reminder` | Bell | lucide-react-native |
| `alarm` | AlarmClockIcon | shared/ui |
| `periodic` | Repeat | lucide-react-native |
| `simple` | — | нет иконки |
| voice (`simple` + payload) | — | нет иконки |

Размер иконки — 11px, совпадает с размером шрифта времени отправки (fontSize: 11).

Иконка располагается в той же строке `metaRow`, слева от времени отправки. Это минимальное изменение, не влияющее на вёрстку.

## Влияние на архитектуру

- **Слой:** `pages/chat-room` — визуальное изменение в `MessageBubble.tsx`
- Импорты иконок из `lucide-react-native` (Bell, Repeat) и `shared/ui` (AlarmClockIcon)
- Паттерн `TYPE_ICON` повторяет существующий в `pages/scheduled/ScheduledItem.tsx`

## Альтернативы

1. **Показывать иконку и для `simple` (Send)** — избыточно, загромождает баббл, большинство сообщений и так simple.
2. **Использовать цветовое кодирование вместо иконок** — менее понятно без легенды.
3. **Показывать иконку вместо текста времени** — теряется информация о времени.

## Оценка сложности

- Время: ~15 мин
- Риски: минимальные — изменение только в одном компоненте, только визуальное
