# Пиксельная контурная аватарка чата

**Статус:** implemented

## Название фичи

`pixel-contour-avatar` — локальная генерация сильно упрощённой пиксельной аватарки из фото: только контуры, с поддержкой цвета (в духе Streamline Pixel / emoji-иконок).

## Описание проблемы

При выборе фото для чата сейчас сохраняется обычный JPEG (`media/avatars/{id}.jpg`). Это выбивается из terminal / pixel-языка UI: иконки чатов уже Streamline Pixel, а фото остаётся фотореалистичным.

Нужно:

1. Автоматически превращать выбранное фото в **пиксельные контуры** (как на референсе: силуэт + глаза/нос/рот, без текстур).
2. **Поддерживать цвета** — не только Ч/Б GameBoy, а ограниченные цвета с оригинала (как «цветной пиксельный эмодзи»).
3. Делать это **локально на устройстве**, быстро и предсказуемо.

Референс: [`docs/design/refs/pixel-contour-avatar-example.png`](../design/refs/pixel-contour-avatar-example.png).

## Предлагаемое решение

### UX

1. Пользователь жмёт «Фото» в `ChatForm`.
2. Системный пикер → изображение (до 512×512).
3. Под капотом сразу строится пиксельная контурная версия.
4. В превью и в списке чатов показывается уже обработанный PNG (прозрачный фон + цветные/моно контуры).
5. Выбор иконки из пака без изменений.

### Алгоритм (ядро)

Вход: RGBA-буфер. Выход: PNG с альфой.

1. Center-crop в квадрат.
2. Уменьшение до рабочего размера (~`pixelGrid * 3`, box/area) — не bilinear «мыло».
3. Luminance + контраст.
4. Sobel → карта краёв; лёгкая дилатация (толще линии, как у 8-bit).
5. Даунскейл до `pixelGrid` (24–40): для ячейки — max edge + средний цвет пикселей с сильным краем.
6. Порог: только контурные ячейки → непрозрачные; остальное — прозрачно.
7. Posterize цвета (`posterizeLevels`, по умолчанию 4–6) — «emoji-like» палитра.
8. Режим `mono`: контуры чёрные; `color` (default): posterize RGB с лёгким затемнением для читаемости.
9. Nearest-neighbor upscale до `outputSize` (128) — крупные квадраты без сглаживания.

### Почему не Nitro в v1

В репозитории нет first-party Nitro-модулей (Nitro только у audio). Полный HybridObject (Swift/Kotlin/C++ + nitrogen) — отдельный объём работ.

Для аватара пикер уже даёт ≤512px; после раннего даунскейла работа идёт на сетке ~32×32. **Чистый TS-пайплайн на этом размере укладывается в десятки мс** и достаточен для save/preview.

Публичный API спроектирован так, чтобы позже заменить реализацию на Nitro C++ без смены call-site:

```ts
createPixelContourAvatar(input: PixelAvatarInput, options?: PixelAvatarOptions): Promise<PixelAvatarResult>
```

Целевой нативный путь (v2, при live-preview / 4K без превью-ресайза): тот же алгоритм в C++, обёртка Nitro `createPixelAvatar(path, options) → path`.

### API / опции

| Опция | Default | Описание |
|-------|---------|----------|
| `pixelGrid` | `32` | Сетка пикселей (16–64) |
| `outputSize` | `128` | Размер PNG |
| `contrast` | `2.2` | Контраст luminance |
| `edgeThreshold` | `0.22` | Порог контура |
| `posterizeLevels` | `5` | Уровней на канал (color) |
| `colorMode` | `'color'` | `'color'` \| `'mono'` |
| `edgeDilate` | `1` | Толщина контура |

### Хранение

- Файл: `media/avatars/{chatId}.png` (альфа обязательна).
- `chats.avatar_path` = относительный путь, как сейчас для фото.
- Иконки пака по-прежнему хранятся как id в том же поле.

## Альтернативы

| Вариант | Плюсы | Минусы | Решение |
|---------|-------|--------|---------|
| Nitro C++ сразу | Макс. скорость | Нет инфраструктуры модуля, долгий setup | Отложить в v2 |
| Skia shaders | GPU, красиво | Тяжёлая зависимость ради одной фичи | Не брать |
| Обычный JPEG без обработки | Уже есть | Не pixel/contour | Отвергнуто |
| Только Ч/Б | Ближе к референсу 1:1 | Нет цвета «как у эмодзи» | Режим `mono` опционален; default `color` |

## Оценка сложности

| Часть | Оценка |
|-------|--------|
| Алгоритм + тесты | S–M |
| Decode JPEG / encode PNG | S |
| Встройка в ChatForm | S |
| Nitro v2 (опционально) | L |

**Риски:** слишком агрессивный порог → «пустой» аватар; слишком слабый → шум. Defaults подобраны под портреты; при необходимости крутить `edgeThreshold` / `pixelGrid`.

## Тестирование

- Unit: crop, sobel/threshold, posterize, mono vs color, upscale NN, прозрачный фон вне контуров.
- Ручное: портрет / объект / тёмный фон; сравнение с референсом по «читаемости» силуэта.
