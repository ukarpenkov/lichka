# Свайп-переключение главных табов

**Статус:** implemented

## Описание проблемы

Главная навигация приложения построена на `@react-navigation/bottom-tabs`. Переключение между табами работает только через тап по иконкам в нижней панели. Пользователи ожидают возможности листать экраны горизонтальным свайпом, как в нативных iOS/Android приложениях.

## Предлагаемое решение

Заменить `createBottomTabNavigator` на кастомный компонент `MainTabs`, который рендерит три экрана в горизонтальном ряду и управляет их положением через `react-native-reanimated` + `react-native-gesture-handler`.

- Перетаскивание за пальцем с жесткой привязкой к ближайшему табу.
- Пружинная анимация переключения с конфигом `SPRING_SNAP` из проекта.
- Тап по иконкам нижней панели также плавно анимирует переход.
- Сохранены вложенные стеки чатов и настроек (`ChatStack`, `SettingsStack`).

## Влияние на архитектуру (FSD)

- `app/AppNavigator.tsx` — заменён `MainTabs` с `bottom-tabs` на кастомный pager.
- `app/SwipeablePager.tsx` — новый виджет-пейджер (можно переиспользовать).
- `app/MainTabsContext.tsx` — контекст видимости таба.
- `app/mainTabsApi.ts` — императивный API для cross-tab переходов.
- `features/notifications` — переход к чату из уведомления теперь через `mainTabsApi`.
- `pages/chat-list`, `pages/scheduled`, `pages/settings` — обновлена логика обновления данных на основе видимости таба.

## Альтернативы

1. Использовать `@react-navigation/material-top-tabs` — дал бы нативный свайп, но не позволил бы разместить панель снизу и тонко контролировать анимацию reanimated.
2. Оставить bottom-tabs и только добавить жест, триггерящий переключение таба — не даст "листания" экранов пальцем.

Выбран кастомный pager, так как он даёт полный контроль над анимацией и соответствует существующей инфраструктуре reanimated.

## Оценка сложности

- Время: ~2-3 часа.
- Риски: нарушение фокуса/обновления экранов при смене табов, конфликты жестов со вложенными экранами. Учтены через `useTabVisible` и отключение свайпа на внутренних экранах.
