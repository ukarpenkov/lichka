# 📔 Journal — оффлайн-дневник в формате «Telegram с самим собой»

## 🧠 Общая идея проекта

Разработать **Android-приложение** для оффлайн-ведения заметок, журналов и личных логов.

### Главная концепция

- приложение ощущается как **«Telegram с самим собой**»;
- пользователь ведёт **тематические журналы**;
- записи создаются в **формате сообщений**;
- **время сообщений нельзя изменять**;
- всё работает полностью **локально без сервера**;
- максимальный акцент на **UX, анимациях, tactile feedback, звуках и эмоциональном опыте** использования.

### Эмоциональный тон приложения

Приложение должно создавать ощущение:

- ✨ **премиальности**;
- 🧠 **продуманности**;
- 🧬 **«живого» интерфейса**;
- 🤝 **глубокого взаимодействия** с пользователем.

## 📦 Структура хранения SQLite (пример)

```sql
-- Журналы
CREATE TABLE journals (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  icon TEXT,
  color TEXT,
  created_at INTEGER NOT NULL
);

-- Сообщения
CREATE TABLE messages (
  id TEXT PRIMARY KEY,
  journal_id TEXT NOT NULL,
  content TEXT NOT NULL,
  created_at INTEGER NOT NULL,
  updated_at INTEGER,
  FOREIGN KEY (journal_id) REFERENCES journals(id) ON DELETE CASCADE
);

-- Общие заметки
CREATE TABLE general_notes (
  id TEXT PRIMARY KEY,
  title TEXT,
  content TEXT,
  is_pinned INTEGER DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER
);

-- Напоминания
CREATE TABLE reminders (
  id TEXT PRIMARY KEY,
  target_id TEXT, -- journal_id или note_id
  target_type TEXT, -- 'journal' или 'note'
  type TEXT, -- 'alarm' или 'push'
  schedule TEXT, -- cron-like или интервал
  enabled INTEGER DEFAULT 1
);



```
