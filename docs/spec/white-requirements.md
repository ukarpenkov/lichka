# Journal (Lichka) — основное ТЗ

> **Статус:** формируется через ответы на вопросы ниже.  
> **Источник:** [draft-requirements.md](./draft-requirements.md)  
> **Цель документа:** гибкое ТЗ для разработки — удобство, скорость, минимум шагов, минимализм в дизайне.

---

## Принципы продукта (зафиксировано)

- **Оффлайн-first** — всё локально, без сервера.
- **«Telegram с самим собой»** — тематические журналы, сообщения по времени; **время можно исправить**, но с **индикатором «изменено»** (#58, #59).
- **Скорость действий** — минимум шагов от намерения до результата.
- **Минимализм UI** — по умолчанию **мягкий белый** фон и чёрный текст; тёмная тема — **полностью чёрный** фон и белый текст; в настройках — **11 пресетов** фона и текста; **формы — text-кнопки без border**; **нижние табы и 4 действия отправки в чате — иконки**; border в MVP нет (точечно — только если позже без него нельзя).
- **Гибкость архитектуры** — готовность к изменениям без переписывания ядра.
- **При конфликте приоритетов:** скорость пользователя и визуальная тишина > декоративность статического UI.

### Список чатов и отправка сообщений

| # | Вопрос | Ответ |
|---|--------|-------|
| 9a | **Список чатов (таб «Чаты»)** — как выглядит элемент списка? | **круглый аватар + название** |
| 9b | **Раскладка списка** | **вертикально, один чат за другим** |
| 9b1 | **Создание нового чата** | **FAB справа снизу** → форма: название + аватар (**галерея** или **emoji Android**); для картинки — **редактор под круг** перед сохранением |
| 9c | **Способы отправки из поля ввода чата** | **4 иконки** внизу экрана чата (не text): сообщение / напоминание / будильник / периодичность (см. ниже) |

**Способы отправки сообщения из чата (кнопки под полем ввода):**

1. **`simple`** (иконка «отправить») — простое сообщение; **сразу в ленте чата**.
2. **`reminder`** (иконка) — напоминание + выбор даты/времени → **push**; до времени — «Запланировано»; **в ленте чата — когда наступает время**.
3. **`alarm`** (иконка) — будильник + выбор даты/времени → **full-screen alarm** (~10 мин или стандартная длительность).
4. **`periodic`** (иконка) — периодичность + интервал → каждое срабатывание через **push**.

**UI выбора даты/времени (будильник и напоминание):**

- **2 концентрических круга:** внешний — **дни**, внутренний — **месяцы**.
- **По центру** — скролл **часов и минут**.
- **Год** — выбор **справа сверху**.

```
                    [ год ▾ ]
         ┌─────────────────────┐
         │    ╭─── дни ───╮    │
         │   ╭── месяцы ──╮    │
         │   │  12 : 30   │    │  ← скролл часов и минут
         │   ╰────────────╯    │
         │    ╰─────────────╯   │
         └─────────────────────┘
```

**UI выбора периодичности:**

- **Заготовки:** каждые 5 / 10 / 15 мин, час, день.
- **Кастомный интервал** — редактируется **внизу экрана** (не только пресеты).

| # | Вопрос | Ответ |

**Стек анимаций MVP (Reanimated 4.x + Gesture Handler):**

| Сценарий (#15) | API Reanimated | Решение для MVP |
|----------------|----------------|-----------------|
| **Появление сообщения** | [Entering/Exiting](https://docs.swmansion.com/react-native-reanimated/docs/layout-animations/entering-exiting-animations) — `FadeIn` / `FadeInUp` + короткий `withTiming` или мягкий `springify()` | **обязательно** — тихое появление пузыря, без bounce |
| **Sticky dates в ленте** | [Layout transitions](https://docs.swmansion.com/react-native-reanimated/docs/layout-animations/layout-transitions) — `LinearTransition` на заголовке даты; `useAnimatedScrollHandler` для скролла | **обязательно** — плавная смена/прилипание, без рывков |
| **Скролл чата / списков** | `useAnimatedScrollHandler`, `useAnimatedStyle` | **обязательно** — инерция и отзывчивость на UI thread |
| **Поле ввода + клавиатура** | [`useAnimatedKeyboard`](https://docs.swmansion.com/react-native-reanimated/docs/device/useAnimatedKeyboard) | **обязательно** — composer не прыгает |
| **Wheel picker (дата/время)** | Gesture Handler (`Pan`) + `useSharedValue` / `withSpring` / `withTiming` | **обязательно** — кастомные круги и центральный скролл |
| **Запись голоса** | Gesture Handler (`LongPress`, `Pan`) + `withSpring` на индикаторе записи | **обязательно** — press/hold/release на UI thread |
| **Home → Journal (аватар)** | [Shared Element Transitions](https://docs.swmansion.com/react-native-reanimated/docs/shared-element-transitions/overview) — `sharedTransitionTag` на аватаре | **желательно, не блокер MVP** — фича **experimental** (feature flag); fallback: обычный stack transition + `withTiming`. Работает только в **native stack** внутри таба «Чаты», не между табами |
| **Переходы табов** | Нативная навигация / лёгкий `withTiming` | **без SET** — tab navigator [не поддерживает](https://docs.swmansion.com/react-native-reanimated/docs/shared-element-transitions/overview) shared transitions |
| **Parallax, blur, gradients** | Кастомные `useAnimatedStyle` | **out of MVP** — против принципа «не броско» (#15) |

**Технические правила:**
- В проекте уже **Reanimated 4.3.1**; добавить **Gesture Handler** для жестов.
- Билдеры анимаций — **вне компонента** или через `useMemo` (perf).
- Учитывать `reduceMotion` для accessibility.
- Shared transitions — только если стабильны на целевых Android; иначе откладываем без потери UX.

| # | Вопрос | Ответ |

- **Только чат** — минимально это лента сообщений внутри чата; без заметок и без global notes в MVP.
- **Модель данных:** **Чат** → **Сообщение**; 4 типа сообщения: `simple` (сразу в ленте), `reminder` (в ленте когда наступит время), `alarm`, `periodic`. **v2+:** заметки с привязкой к чату.
- **Запуск приложения:** всегда **таб «Чаты»** (список журналов), не последний открытый чат.
- **Нижняя навигация (3 таба):** **Чаты** | **Запланировано** | **Настройки** — **иконки**, не text-лейблы.
- **Чаты:** вертикальный список журналов; элемент = **круглый аватар + название** (аватар: фото / emoji / fallback — первая буква названия). **Глобальный поиск** по всем сообщениям — на этом экране.
- **Создание чата:** FAB справа снизу → форма (название + аватар: **галерея или emoji Android**); для картинки — редактор под круг. **Без цвета журнала** — см. #6.
- **Запланировано:** единый список напоминаний и будильников, **отсортированный по времени**; тап по элементу → переход в чат к исходному сообщению.
- **Экран чата:** поле ввода + **4 иконки отправки:** сообщение / напоминание / будильник / периодичность. **Поиск по чату** — только внутри открытого чата.
  - заметка → иконка **сообщение**;
  - напоминание → иконка **напоминание** + picker даты/времени → **push**;
  - будильник → иконка **будильник** + picker даты/времени → **full-screen alarm**;
  - периодичность → иконка **периодичность** + пресеты (5/10/15 мин, час, день) + кастом внизу → **push**.
- **Picker даты/времени:** 2 круга (дни / месяцы), центр — часы и минуты, год — справа сверху.
- **Тема:** light/dark по умолчанию + кастомизация цвета фона и текста в настройках.
- **Без шифрования** — ни export, ни БД at rest в MVP v1.

### UX Principles

- **Нижняя навигация:** 3 таба — **Чаты**, **Запланировано**, **Настройки** — **иконки** (не text); без отдельного таба заметок.
- **Список чатов (таб «Чаты»):** минималистичный вертикальный список (круглый аватар + название); это **стартовый экран** при каждом запуске. **Глобальный поиск** по всем сообщениям — здесь. Аватар — фото, emoji или **первая буква названия** (монохром); **цвет журнала не используем**.
- **Создание чата:** FAB справа снизу на табе «Чаты» → inline-форма (название, аватар: **картинка из галереи или emoji Android**, редактор под круг только для картинки); отдельный экран не нужен.
- **Запланировано:** агрегированный хронологический список всех напоминаний и будильников; тап открывает чат с привязанным сообщением.
- **Ввод сообщений только внутри чата** — без FAB для быстрой записи; сначала выбор чата, затем поле ввода и **4 иконки отправки**.
- **Один экран ввода** — тип отправки выбирается **отдельной иконкой** (сообщение / напоминание / будильник / периодичность); канал доставки определяется типом (напоминание и периодичность → push, будильник → alarm).
- **Поиск:** глобальный — на табе «Чаты» (по всем сообщениям); локальный — внутри открытого чата.
- **Picker даты/времени:** концентрические круги (дни снаружи, месяцы внутри) + скролл часов/минут по центру + год справа сверху.
- **Периодичность:** пресеты 5/10/15 мин, час, день; кастомный интервал редактируется внизу экрана.
- **Кнопки и иконки:**
  - **Text-кнопки без border** — формы и второстепенные действия; primary/secondary не различаем в MVP.
  - **Иконки** — нижние табы; **4 действия отправки в чате** (сообщение, напоминание, будильник, периодичность); системные (назад, микрофон, камера, FAB).
  - **Border** — не используем в MVP; точечно — только если позже без него нельзя.
  - **Touch targets:** composer чата (ввод, 4 иконки, голос) — приоритет скорости, generous hit area; toolbar/text-кнопки — компактнее (#20).
- **Тема и цвета:**
  - **Light (default):** белый фон, чёрный текст.
  - **Dark:** чёрный фон, белый текст.
  - **Настройки:** выбор из **11 пресетов** фон→текст (см. #5).
  - Material You / dynamic color — **опционально** в настройках, только если SDK даёт пару фон + текст; иначе — пресеты и кастом (#5, #19).

### Data Model

**MVP — две сущности контента:**

```
chats
  id, title, avatar_path (relative), created_at, …

messages
  id, chat_id (FK), type, body,
  scheduled_at, interval_minutes, enabled,   -- nullable; для reminder | alarm | periodic (#26)
  payload (JSON), metadata (JSON, опц.), created_at, updated_at, …
  type ∈ { simple, reminder, alarm, periodic }
```

| `type` | В ленте чата | «Запланировано» | Доставка |
|--------|--------------|-----------------|----------|
| `simple` | **сразу** | — | — |
| `reminder` | **когда наступит время** | до времени | push |
| `alarm` | **когда наступит время** | до времени | full-screen alarm |
| `periodic` | **при каждом срабатывании** | пока активно | push по интервалу |

- **General Notes / global notes** — не входят в MVP v1. **v2+:** только заметки, **прикреплённые к чату**.
- **Редактирование текста:** только последняя версия + `updated_at`; история правок не хранится (#22).
- **Удаление:** hard delete — запись удаляется из БД без `deleted_at` (#23).
- Напоминания не «висят в воздухе» — это **сообщения**; расписание для запросов — в **колонках** (`scheduled_at`, …), детали по `type` — в **`payload`** (#26).
- **JSON vs таблицы (#26):** колонки для поиска и планировщика; JSON для вариативности; отдельные таблицы — attachments и 1:N, не «всё в JSON».

**Медиа и файлы (#31, #32):**

```
media/
  avatars/{chatId}.jpg      — avatar_path в chats
  voice/{messageId}.m4a     — post-MVP / v1.1 attachments
  images/{messageId}.jpg    — post-MVP attachments
```

- В БД — **относительный** путь; чтение через `resolveMediaPath()` в `shared/`.
- **Hard delete** (#23) — удалять файлы записи; **startup cleanup** — удалять orphan без ссылки в БД.
- **Изображения (#33):** один compressed на запись; original не храним; resize при импорте (avatars ~512 px; message images ~1200–1600 px post-MVP).

**Миграции схемы (#25):**

```
shared/db/migrations/
  001_initial.sql      — chats, messages, schema_migrations
  002_fts5.sql         — FTS5 для поиска (#36)
  003_notes.sql        — v2+: notes с chat_id
```

- При первом открытии БД runner сравнивает `MAX(version)` в `schema_migrations` с доступными файлами и применяет недостающие.
- Каждая миграция — **одна транзакция**; старые файлы **не редактируем**, только добавляем новые.
