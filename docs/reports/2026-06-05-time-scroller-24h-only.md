# TimeScroller: только 24-часовой формат

**Дата:** 2026-06-05
**Промпт/задача:** Убрать AM/PM из TimeScroller, оставить только 24-часовой формат для всех

## Что сделано

- Удалена функция `getIs24Hour()`, определявшая формат через `Intl.DateTimeFormat`
- Удалён `useRef(getIs24Hour()).current` — мемоизированный флаг `is24`
- Упрощён `formatHour` — теперь всегда возвращает `"00"`–`"23"` без ветвления на 12/24
- Удалён блок отображения AM/PM лейбла справа от колонки минут

## Изменённые файлы

- `src/widgets/datetime-picker/TimeScroller.tsx` — удалены функция `getIs24Hour`, ref `is24`, логика 12-часового формата и AM/PM лейбл

## Принятые решения

- Формат **всегда 24-часовой**, без зависимости от локали устройства
- Внутреннее состояние часов (0–23) не изменилось — меняется только отображение
- Настройка формата времени в `AppSettings` не добавляется — формат захардкожен

## Известные ограничения

- Пользователи в локалях с 12-часовым форматом (en-US) больше не увидят AM/PM — принято как требование
- `formatScheduledAt` в `dateUtils.ts` всё ещё использует `toLocaleTimeString` и может показывать 12-часовой формат в других местах приложения (не в пикере)

## Изменения в требованиях

| Было | Стало |
|------|-------|
| Формат часов определяется локалью устройства (`Intl.DateTimeFormat`) | Формат всегда 24-часовой |
| AM/PM лейбл отображается при 12-часовом формате | AM/PM удалён полностью |
| `getIs24Hour()` + `is24` ref для переключения | Удалены, `formatHour` возвращает всегда `"00"`–`"23"` |

## Тестирование

- Ручная проверка: TimeScroller показывает часы 00–23, AM/PM отсутствует
- Регрессии: логика выбора часов/минут (scroll, press, snap) не затронута
