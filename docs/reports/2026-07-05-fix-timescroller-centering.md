# Исправление центрирования выбранных значений в TimeScroller

**Дата:** 2026-07-05
**Промпт/задача:** После остановки прокрутки пикера времени выбранные часы и минуты не выравниваются строго по центру голубого прямоугольника выделения. Значения остаются со смещением.

## Что сделано
- Убрано переключение `fontSize` (22 → 28) при выделении элемента, которое являлось причиной смещения
- Установлен единый `fontSize: 24` для всех элементов (выделенных и невыделенных)
- Добавлен `lineHeight: ITEM_HEIGHT` (46) — лайнбокс текста совпадает с высотой ячейки, что гарантирует точное центрирование через `justifyContent: 'center'`
- Выделение выбранного значения теперь через `fontWeight: '700'` (было '600') и цвет

## Причин бага
Разные `fontSize` (22 vs 28) имеют разные внутренние метрики шрифта (ascender, descender, leading). Даже при `justifyContent: 'center'` на контейнере фиксированной высоты, визуальный центр текста сдвигался при смене размера. Особенно заметно на Android, где React Native не всегда точно центрирует текст разных размеров в одном контейнере.

## Изменённые файлы
- `src/widgets/datetime-picker/TimeScroller.tsx` — стили `renderItem`: fontSize, lineHeight, fontWeight

## Принятые решения
- Единый fontSize вместо двух — самое надёжное решение, не зависит от платформенных особенностей центрирования текста
- `lineHeight: ITEM_HEIGHT` — дополнительно страхует центрирование, лайнбокс точно совпадает с ячейкой
- `fontWeight: '700'` вместо '600' — более заметное выделение без изменения размера

## Известные ограничения
- Визуальный акцент на выбранном элементе теперь только через вес и цвет (без увеличения размера)

## Тестирование
- Открыть пикер → прокрутить часы/минуты → после остановки значения должны быть строго по центру голубого прямоугольника
- Проверить на обоих списках (часы и минуты)
- Проверить тап на конкретное значение — должно центрироваться
