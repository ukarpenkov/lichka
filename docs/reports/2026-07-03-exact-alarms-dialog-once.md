# Исправление повторяющегося диалога Exact alarms при создании будильника

**Дата:** 2026-07-03
**Промпт/задача:** Завести баг и исправить: диалог Exact alarms показывается при каждом создании будильника вместо одного раза.

## Что сделано

- Сохранение флага `alarm_guide_shown` в AsyncStorage при первом показе диалога с инструкциями по настройке разрешений (Exact alarms, Battery optimization, Show over lock screen, Auto-start)
- При последующих созданиях будильника флаг проверяется — если уже был показан, диалог пропускается, и будильник создаётся сразу (с проверкой permission)
- Флаг записывается до показа диалога — даже если пользователь закроет диалог без нажатия «Готово» или приложение упадёт, повторно диалог не появится

## Изменённые файлы

- `src/widgets/message-composer/MessageComposer.tsx` — добавлен импорт AsyncStorage; в `handlePickerConfirm` проверка флага перед показом alarm-guide диалога, установка флага при первом показе
- `docs/bugs/exact-alarms-dialog-repeated.md` — баг-репорт

## Принятые решения

- Флаг сохраняется в AsyncStorage (ключ `alarm_guide_shown`), а не в SQLite-таблице `settings` — флаг не является пользовательской настройкой, это внутреннее состояние UI
- AsyncStorage уже был в зависимостях проекта (`@react-native-async-storage/async-storage: ^3.1.0`), ранее не использовался в коде
- Флаг устанавливается ДО показа диалога, чтобы гарантировать однократный показ даже при аварийном завершении

## Известные ограничения

- Permission dialog (fallback при отсутствии `SCHEDULE_EXACT_ALARM`) по-прежнему показывается при каждой попытке создать будильник, если permission нет — это функциональное сообщение, а не информационный гайд
