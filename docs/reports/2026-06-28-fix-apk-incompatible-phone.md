# Fix: APK not installed — incompatible with phone

**Дата:** 2026-06-28
**Промпт/задача:** APK не устанавливается на телефон с ошибкой "Приложение не установлено, так как оно несовместимо с вашим телефоном"

## Что сделано
- Изменена архитектура сборки в `android/gradle.properties`

## Изменённые файлы
- `android/gradle.properties` — `reactNativeArchitectures` изменён с `x86_84` на `arm64-v8a,armeabi-v7a`

## Причина
В коммите `90b0c25` (optimize Gradle build performance) было выставлено `reactNativeArchitectures=x86_64` для ускорения сборки на эмуляторе. APK собирался только для x86_64 архитектуры, которая используется эмуляторами на x86. Реальные телефоны работают на ARM (arm64-v8a / armeabi-v7a), поэтому APK отвергался системой как несовместимый.

## Исправление
`reactNativeArchitectures=arm64-v8a,armeabi-v7a` — покрывает все современные Android-устройства.

## Известные ограничения
- Сборка будет медленнее (сборка для 2 архитектур вместо 1)
- Для development можно вернуть `x86_64` или `arm64-v8a` (только нужная архитектура)

## Тестирование
- Пользователь должен пересобрать APK и установить на телефон
