# Исправление автофокуса поиска в списке чатов

**Дата:** 2026-07-02
**Промпт/задача:** Исправить баг — клавиатура не открывается автоматически при активации поиска в списке чатов

## Что сделано
- Убран `InteractionManager.runAfterInteractions`, откладывавший фокус на поле поиска
- Добавлен `autoFocus` на `<TextInput>` для мгновенного фокуса при монтировании
- Оставлен `useEffect` для сброса состояния при скрытии поиска

## Изменённые файлы
- `src/pages/chat-list/GlobalSearch.tsx` — `InteractionManager` → `autoFocus`

## Принятые решения
- `autoFocus` надёжнее ручного `.focus()` через `InteractionManager`, т.к. React Native обрабатывает его на уровне нативного события фокуса при монтировании
- `inputRef` сохранён для возможного использования в будущем
- `SearchOverlay.tsx` (поиск внутри чата) не трогали — там уже был прямой вызов `.focus()` без `InteractionManager`

## Известные ограничения
- На некоторых Android-клавиатурах (Gboard) может быть задержка при первом открытии — это особенность ОС, не влияющая на поведение фокуса

## Тестирование
- Визуальная проверка: поле поиска получает фокус, клавиатура открывается сразу после нажатия на иконку поиска
