# Исправление пустого экрана при входе в чат

**Дата:** 2026-06-28
**Промпт/задача:** Когда заходим в чат на устройстве, видим пустой экран с placeholder-иконкой — чат не отображается, надо исправить

## Что сделано
- Изменён тип состояния `chat` с `Chat | null` на `Chat | null | undefined`
  - `undefined` = данные ещё загружаются (показываем ActivityIndicator)
  - `null` = чат не найден в БД (показываем текст «Чат не найден»)
  - объект `Chat` = чат найден (показываем контент)
- Добавлена индикация загрузки (`ActivityIndicator`) вместо пустого экрана на первом рендере
- Добавлено сообщение об ошибке «Чат не найден» если чат не существует в БД
- Добавлен ключ локализации `chatNotFound` (ru/en)

## Изменённые файлы
- `src/pages/chat-room/ChatRoomScreen.tsx` — три состояния рендеринга (loading / not found / content)
- `src/shared/config/locale.ts` — ключ `chatNotFound` для ru и en

## Принятые решения
- Использовать `undefined` для состояния загрузки — стандартный React-паттерн для отличия «ещё не загрузили» от «загрузили, но ничего нет»
- `loadData()` явно приводит результат `getChatById()` к `null` через `?? null`, чтобы чётко отделить «не нашлось» от «ещё грузим»

## Известные ограничения
- Нет кнопки «Назад» на экране «Чат не найден» — пользователь должен использовать системную кнопку back

## Тестирование
- TypeScript компилируется без новых ошибок (существующие ошибки в voice player/recorder не связаны)
