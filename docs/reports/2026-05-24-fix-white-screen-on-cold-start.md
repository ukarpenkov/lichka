# Исправление белого экрана при холодном запуске

**Дата:** 2026-05-24
**Промпт/задача:** При запуске приложения отображается белый экран вместо ChatListScreen

## Что сделано
- Вызов `runMigrations()` перенесён из `useEffect` в тело модуля `App.tsx` (выполняется один раз при импорте модуля, до любого рендера)

## Изменённые файлы
- `App.tsx` — `runMigrations()` вызывается на уровне модуля, `useEffect` удалён

## Принятые решения
- **Корень проблемы:** React выполняет `useEffect` снизу вверх (children first). `ThemeProvider` (child) свой эффект выполнял раньше `App` (parent), и запрос к таблице `settings` падал, потому что миграция 3 ещё не была применена
- **Решение:** синхронный вызов `runMigrations()` при загрузке модуля гарантирует, что БД готова до монтирования любого компонента

## Известные ограничения
- `runMigrations()` блокирует JS-поток на старте. Для миграций с большим объёмом данных потребуется асинхронный вариант с экраном загрузки

## Тестирование
- Холодный запуск (чистая БД) — экран "Создайте первый чат" отображается корректно
- Повторный запуск (миграции уже применены) — экран отображается без изменений
