# Запрос разрешения на уведомления при запуске

**Дата:** 2026-06-28
**Промпт/задача:** Добавить запрос разрешения на уведомления при запуске приложения

## Что сделано
- Добавлен вызов `requestNotificationPermission()` в `AppInitProvider.tsx` во время инициализации приложения
- Запрос разрешения происходит параллельно с другими задачами инициализации (миграции БД, регистрация каналов)
- На Android 13+ (API 33) при первом запуске показывается системный диалог запроса разрешения

## Изменённые файлы
- `src/app/AppInitProvider.tsx` — добавлен импорт и вызов `requestNotificationPermission()`

## Принятые решения
- Запрос разрешения во время инициализации, а не при первом уведомлении — чтобы система знала предпочтение пользователя заранее
- Ошибка запроса игнорируется (`.catch(() => {})`) — приложение продолжает работу, если разрешение не получено

## Известные ограничения
- На iOS запрос разрешения обрабатывается через entitlement, не через runtime permission

## Тестирование
- TypeScript компилируется без ошибок (`npx tsc --noEmit`)
