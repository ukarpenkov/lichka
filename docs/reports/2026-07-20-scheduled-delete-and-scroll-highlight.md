# Удаление и навигация к запланированным сообщениям

**Дата:** 2026-07-20
**Промпт/задача:** Long-press удаление записи из «Запланировано» (с удалением из чата); тап открывает чат, скроллит к сообщению и кратко подсвечивает его.

## Что сделано
- Long-press на элементе списка «Запланировано» открывает подтверждение удаления; при подтверждении сообщение удаляется из БД и отменяется связанное уведомление — запись пропадает и из чата.
- Тап по запланированному по-прежнему открывает чат с `messageId`; чат скроллит к сообщению и подсвечивает пузырёк ~1 секунду.
- Будущие reminder/alarm и ещё не сработавшие periodic временно подмешиваются в ленту чата при переходе по `messageId`, чтобы скролл/подсветка работали.
- Та же подсветка при выборе сообщения из поиска внутри чата.

## Изменённые файлы
- `src/pages/scheduled/ScheduledItem.tsx` — `onLongPress`, haptic, `delayLongPress={300}`
- `src/pages/scheduled/ScheduledScreen.tsx` — диалог удаления, `deleteMessage` + `cancelNotification`
- `src/pages/chat-room/ChatRoomScreen.tsx` — inject целевого сообщения, scroll once, `highlighted` на `MessageBubble`

## Принятые решения
- Удаление через `AlertDialog` сразу по long-press (без промежуточного context menu) — соответствует запросу «зажать и удалить».
- Подсветка через уже существующий prop `MessageBubble.highlighted` (~18% ink overlay), длительность 1 с.
- Не меняли общую видимость будущих сообщений в ленте — только точечный inject при навигации с `messageId`.

## Известные ограничения
- После ухода из чата будущие сообщения снова скрыты из ленты до наступления времени (как и раньше).
- Коммит не создавался — в правилах пользователя коммит только по явному запросу.

## Тестирование
- Ручная проверка: long-press → confirm → запись исчезает из «Запланировано» и из чата.
- Ручная проверка: тап по будущему reminder/alarm → чат открывается, скролл к сообщению, краткая подсветка.
- Линтер по изменённым файлам — без ошибок.
