# Исправление автофокуса поиска в списке чатов

**Дата:** 2026-07-02
**Промпт/задача:** Исправить баг — клавиатура не открывается автоматически при активации поиска в списке чатов

## Что сделано
- Добавлен `InteractionManager.runAfterInteractions` в `GlobalSearch.tsx` для отложенного вызова `focus()` до завершения анимации `SlideInDown`

## Изменённые файлы
- `src/pages/chat-list/GlobalSearch.tsx:2` — добавлен импорт `InteractionManager`
- `src/pages/chat-list/GlobalSearch.tsx:28-33` — вызов `focus()` обёрнут в `InteractionManager.runAfterInteractions`

## Принятые решения
- Использован `InteractionManager.runAfterInteractions` вместо `setTimeout`, так как это идиоматичный для React Native способ дождаться завершения анимаций и нативных переходов
- Не использован `autoFocus` на `TextInput` — он ненадёжен при условном рендеринге с анимацией
- `SearchOverlay.tsx` (чатрум) оставлен без изменений, так как баг зарепорчен только для списка чатов

## Известные ограничения
- На очень медленных устройствах `InteractionManager` может вызвать фокус до завершения анимации входа; при необходимости можно увеличить надёжность дополнительным `setTimeout` с запасом

## Тестирование
- Визуально: при нажатии на иконку поиска поле ввода получает фокус и клавиатура открывается без дополнительного тапа
