# Каналы уведомлений и голос (foreground-only)

**Дата:** 2026-05-24
**Промпт/задача:** Уточнить виды уведомлений (#44): внутренний push + будильник; recording не нужен; голос — как голосовое сообщение, только при запущенном приложении.

## Что сделано
- Зафиксирован ответ на **#44**: 2 канала — `reminders` (push для reminder/periodic) и `alarms` (full-screen будильник).
- Зафиксирован ответ на **#45**: foreground notification и FGS для записи **не нужны** — голос только в foreground.
- Обновлён **#39**: снята привязка FGS к голосу в v1.1; Kotlin для записи не требуется.

## Изменённые файлы
- `docs/spec/white-requirements.md` — ответы #39, #44, #45 и changelog.

## Принятые решения
- **Два канала уведомлений в MVP:** push (`reminders`) и будильник (`alarms`).
- **Без каналов** `recording` и `backup` (backup — post-MVP).
- **Голос** моделируется как голосовое сообщение в чате (press/hold), не как диктофон с фоновой записью.
- При сворачивании/убийстве приложения запись прерывается — осознанное ограничение, упрощает Android-стек.

## Известные ограничения
- Длинная запись голоса невозможна при переключении на другое приложение.
- Если позже понадобится фоновая запись — потребуются FGS + канал `recording` (отдельное решение).

## Тестирование
- Не применимо (только спецификация).
