# Поиск по чату: шрифт плейсхолдера и метки дат

**Дата:** 2026-07-20
**Промпт/задача:** Плейсхолдер «Поиск по чату...» — обычный (system) шрифт, не mono; метки дат в поиске только при совпадениях, с группировкой по дням.

## Что сделано
- Sticky date-сепаратор чата больше не показывается поверх `SearchOverlay` (из-за него при открытии поиска висела «9 июля» без результатов).
- В результатах поиска по чату — группировка по дням через `DateSeparator`: пустой запрос / «ничего не найдено» → без меток; совпадения в разные дни → отдельная метка на каждый день.
- Поле поиска явно на system sans (Android `sans-serif`, iOS — системный default), не JetBrains Mono.
- Unit-тесты на `buildSearchListItems`.

## Изменённые файлы
- `src/pages/chat-room/SearchOverlay.tsx` — группировка результатов, system font у input
- `src/pages/chat-room/ChatRoomScreen.tsx` — скрытие sticky date при `searchVisible`
- `src/pages/chat-room/__tests__/SearchOverlay.test.ts` — тесты группировки
- `docs/reports/2026-07-20-chat-search-placeholder-date-labels.md` — этот отчёт

## Принятые решения
- Метка даты в поиске = тот же `DateSeparator`, что в ленте чата (`── 9 июля ──`).
- Перед группировкой результаты сортируются по `created_at` DESC, чтобы FTS rank не перемешивал дни.
- Исключение из terminal mono-контракта только для search input/placeholder — по явному запросу.

## Известные ограничения
- GlobalSearch («Поиск по сообщениям») не менялся — задача про поиск внутри чата.
- На iOS отдельный `placeholderFontFamily` недоступен: system sans применяется и к введённому тексту.

## Тестирование
- Unit: `buildSearchListItems` — пустой список, два дня, один день, сортировка DESC
- Ручная проверка: открыть поиск → нет sticky «N июля»; найти «привет» за 2 дня → 2 метки + 2 строки
