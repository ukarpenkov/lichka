**Дата:** 2026-05-23

## Изменённые файлы

## Принятые решения

- Интеграция **опциональная**, post-MVP; локальные данные остаются первичными.
- Односторонний экспорт в Calendar (создание/обновление по запросу), без полного импорта событий на первом этапе.
- Scope `calendar.events` запрашивается только при включении интеграции.
- Связь локальной записи с `calendarEventId` для избежания дубликатов и поддержки обновления.

## Известные ограничения

- Детальный UX-поток (экран быстрого ввода, пресеты времени) не проработан — отложено на отдельный design/proposal.
- Не определено, нужен ли отдельный proposal в `docs/features/` до реализации (по правилам проекта — да, при старте разработки).

## Тестирование

- Не применимо (изменения только в спецификации).
