# Cursor rules по development-and-release-rules

**Дата:** 2026-05-22
**Промпт/задача:** Создать правила Cursor на основе `docs/rules/development-and-release-rules.md`, добавить обязательные отчёты в `docs/reports/`

## Что сделано

- Создана директория `.cursor/rules/` с 5 правилами проекта
- Создана директория `docs/reports/` для отчётов по промптам и изменениям кода
- Правило отчётов: каждый промпт и каждое изменение кода сопровождается md-файлом в `docs/reports/`

## Изменённые/созданные файлы

| Файл | Назначение |
|------|------------|
| `.cursor/rules/lichka-reports.mdc` | Обязательные отчёты (always apply) |
| `.cursor/rules/lichka-fsd-architecture.mdc` | FSD: слои, зависимости, public API |
| `.cursor/rules/lichka-commits.mdc` | Conventional Commits, атомарность |
| `.cursor/rules/lichka-feature-proposals.mdc` | Proposal-документы для новых фич |
| `.cursor/rules/lichka-testing.mdc` | Требования к тестам (80%, типы, стиль) |
| `docs/reports/.gitkeep` | Папка для отчётов |

## Принятые решения

- Правила разбиты на 5 файлов (одна тема = один `.mdc`) — рекомендация Cursor для читаемости
- `alwaysApply: true` для отчётов, FSD и коммитов — базовый workflow на каждую сессию
- Тестирование и feature-proposals — `globs`, срабатывают при работе с соответствующими файлами
- Именование отчётов: `YYYY-MM-DD-<краткое-описание>.md`

## Известные ограничения

- FSD-слои в коде ещё не заведены (проект на стадии инициализации, `App.tsx` в корне)
- Правила коммитов в Cursor не заменяют user rule «коммит только по запросу пользователя» — при конфликте приоритет у явного запроса в чате

## Тестирование

- Не применимо (только конфигурация правил и документация)
