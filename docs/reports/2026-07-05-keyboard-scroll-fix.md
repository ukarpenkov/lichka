# Фикс: прокрутка к последнему сообщению при открытии клавиатуры

**Дата:** 2026-07-05
**Промпт/задача:** Баг — нижняя часть последнего сообщения перекрывается при открытой клавиатуре

## Что сделано

- Добавлен слушатель `keyboardDidShow` в `ChatRoomScreen`, который скроллит FlatList в конец при открытии клавиатуры
- Задержка 100мс перед `scrollToEnd` — даёт время на пересчёт layout после изменения `paddingBottom` в `chatAreaAnimatedStyle`
- Блок сообщений и его отступы от клавиатуры (`chatAreaAnimatedStyle`, `containerAnimatedStyle` в композиторе) не изменены

## Изменённые файлы

- `src/pages/chat-room/ChatRoomScreen.tsx` — добавлен `useEffect` со слушателем `keyboardDidShow` (строки 159–166)

## Принятые решения

- Использован `setTimeout(100мс)` вместо `requestAnimationFrame` — layout пересчитывается после изменения `paddingBottom` через Reanimated, нужна гарантия что native layout уже обновлён
- Скролл всегда animated — визуально приятнее чем резкий прыжок

## Известные ограничения

- На iOS системный keyboard avoidance может работать иначе, но `keyboardDidShow` срабатывает на обеих платформах

## Тестирование

- Открыть чат → нажать на поле ввода → последнее сообщение должно быть полностью видно
- Закрыть клавиатуру → поведение без изменений
- Новые сообщения → автоскролл вниз работает как раньше
