# Fix: запрет пролистывания чата после последнего сообщения при открытой клавиатуре

**Дата:** 2026-07-12
**Промпт/задача:** Баг — когда клавиатура открыта, можно пролистать чат ниже последнего сообщения, после которого остаётся пустое место.

## Что сделано

1. Добавлен проп `bounces={!keyboardOpen}` на FlatList — отключает iOS bounce при открытой клавиатуре
2. Добавлен проп `overScrollMode={keyboardOpen ? 'never' : 'auto'}` — отключает Android overscroll при открытой клавиатуре

## Изменённые файлы

- `src/pages/chat-room/ChatRoomScreen.tsx` — добавлены `bounces` и `overScrollMode` на `AnimatedFlatList` (строки 402–403)

## Принятые решения

1. **`bounces` только при открытой клавиатуре** — когда клавиатура закрыта, bounce сохраняется для естественного ощущения прокрутки. При открытой клавиатуре bounce убирается, чтобы пользователь не мог прокрутить в пустую область ниже последнего сообщения.

2. **`overScrollMode` для Android** — на Android аналогичная проблема решается через `overScrollMode="never"`, так как `bounces` — iOS-only проп.

## Известные ограничения

- При закрытой клавиатуре overscroll остаётся разрешённым (по умолчанию).

## Тестирование

- Открыть чат, открыть клавиатуру → прокрутка останавливается на последнем сообщении, пустого места ниже нет
- Закрыть клавиатуру → bounce/overscroll работает как обычно
- Проверить на iOS и Android
