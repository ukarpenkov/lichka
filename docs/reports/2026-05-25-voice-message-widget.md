# Виджет голосового сообщения (VoiceMessage)

**Дата:** 2026-05-25
**Промпт/задача:** Реализация виджета VoiceMessage (задача 5.7)

## Что сделано

- Создан виджет `src/widgets/voice-message/` с компонентом для воспроизведения голосовых сообщений в ленте чата
- Реализован хук `useVoicePlayer` для управления воспроизведением (play/pause/stop) через `react-native-audio-recorder-player`
- Механизм "один играет — остальные стоп": модульный реестр `activeStopFns` останавливает предыдущий плеер при запуске нового
- Компонент `WaveformBar` — визуализация волны из 32 SVG-подобных полосок с анимированным прогрессом через Reanimated
- `VoiceMessage` — основной компонент: кнопка Play/Pause, волна, отображение длительности (остаток при воспроизведении, общее в покое)
- Интеграция в `MessageBubble`: определение голосовых сообщений по `payload.uri` и условный рендеринг `VoiceMessage` вместо текста
- Экспорт через виджет-баррел

## Принятые решения

- **Per-message player** — каждый экземпляр `VoiceMessage` создаёт свой `AudioRecorderPlayer` через хук. Синхронизация через модульный массив `activeStopFns` — при запуске нового плеера все остальные останавливаются.
- **Волна** — 32 полоски с детерминированными псевдо-случайными высотами (seed от `message.id`). Прогресс реализован через два слоя: фоновый (неигранный) и foreground с `overflow: 'hidden'` + animated width.
- **Длительность** — парсится из `message.body` (regex `[Голосовое Xс]`), т.к. реальная длительность не хранится отдельно. При воспроизведении показывается остаток, в покое — общая.
- **Определение голосового сообщения** — по наличию `payload` с JSON `{ uri: "...voice..." }`.
- **Без фонового воспроизведения** — плеер останавливается при unmount компонента (уход со страницы чата).

## Известные ограничения

- Волна статическая (псевдо-случайные высоты), не отражает реальную аудио-амплитуду
- Длительность парсится из текста `body`, а не из отдельного поля — хрупко при изменении формата
- Нет seek (перемотки) по волне — только play/pause

## Тестирование

- Запись голосового сообщения → отображается как waveform-бабл вместо `[Голосовое Xс]`
- Tap play → воспроизведение + анимация волны + обратный отсчёт
- Tap pause → пауза, волна замирает
- Tap play на другом сообщении → первое останавливается, второе начинается
- TypeScript: предсуществующие ошибки (tsconfig `--jsx`), новых нет
