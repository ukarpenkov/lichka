# Правки DateTimePicker — вращение колец и вертикальный скролл времени

**Дата:** 2026-05-26
**Промпт/задача:** Исправить DateTimePicker: кольца дат/месяцев должны вращаться, время — вертикальный скролл

## Что сделано

### DayRing.tsx — вращение вместо выбора по позиции касания
- Убрана обработка жестов из компонента
- Компонент теперь принимает `rotation: SharedValue<number>` и визуально вращает SVG-кольцо через `useAnimatedStyle`
- Статичная дуга-индикатор сверху (не вращается) — показывает выбранный сектор
- Кольцо с днями 1–31, радиус 130px, ширина 40px

### MonthRing.tsx — вращение для месяцев
- Аналогично DayRing: `rotation` SharedValue, `useAnimatedStyle`, статичный индикатор
- 12 меток месяцев (Янв–Дек), радиус 80px, ширина 32px
- `pointerEvents="none"` — не перехватывает касания, жесты обрабатываются родителем

### TimeScroller.tsx — вертикальные скроллы
- Два вертикальных `FlatList` (часы слева, минуты справа) с `snapToInterval={40}`
- Формат 12/24 часа по локали устройства (`Intl.DateTimeFormat`)
- AM/PM индикатор для 12-часового формата
- `onMomentumScrollEnd` + `onScrollEndDrag` — снап к ближайшему значению
- Нажатие на элемент — мгновенная прокрутка к нему

### DateTimePicker.tsx — единый обработчик жестов
- `Gesture.Manual()` на всю область колец
- В `onTouchesDown`: проверка дистанции от центра — если попали в кольцо дней (r 110–150) → активация для дней, если в кольцо месяцев (r 64–96) → активация для месяцев, иначе `manager.fail()` (центр — скролл времени)
- В `onTouchesMove`: расчёт дельты угла и обновление `dayRotation` / `monthRotation`
- В `onTouchesUp`: снап к ближайшему сегменту через `withSpring`
- Начальные повороты колец вычисляются из текущей даты: день D → `-(D-1) * 360/31`, месяц M → `-M * 30`

### Экспорты
- DayRing и MonthRing экспортируют константы (`DAY_RING_INNER`, `MONTH_RING_OUTER` и т.д.) для использования в DateTimePicker

## Изменённые файлы
- `src/widgets/datetime-picker/DayRing.tsx` — полная переработка (визуальный компонент с вращением)
- `src/widgets/datetime-picker/MonthRing.tsx` — полная переработка (визуальный компонент с вращением)
- `src/widgets/datetime-picker/TimeScroller.tsx` — переход на вертикальные скроллы
- `src/widgets/datetime-picker/DateTimePicker.tsx` — единый Gesture.Manual, управление вращениями
- `src/widgets/datetime-picker/circularMath.ts` — без изменений
- `src/widgets/datetime-picker/YearPicker.tsx` — без изменений
- `src/widgets/datetime-picker/index.ts` — без изменений

## Принятые решения
- **Gesture.Manual вместо Pan** — чтобы корректно разделять зоны касания (центр = время, кольца = дата/месяц) через `manager.fail()`/`manager.activate()`
- **Вращение всего SVG через Animated.View** — даёт плавную 60fps анимацию без перерендера SVG-элементов
- **Статичная дуга-индикатор** вместо динамической подсветки номера — упрощает рендеринг, номер под индикатором и есть выбранный
- **Вертикальные FlatList для времени** — стандартный паттерн iOS/Android пикеров времени, интуитивно понятен

## Известные ограничения
- На эмуляторе не тестировалось — требуется запуск и проверка жестов
- При быстром вращении кольца возможна задержка обновления selectedDay/month (runOnJS)

## Тестирование
- TypeScript проходит без ошибок для всех файлов datetime-picker
- Требуется ручное тестирование на эмуляторе:
  - Вращение внешнего кольца (дни) по и против часовой
  - Вращение внутреннего кольца (месяцы)
  - Вертикальный скролл часов и минут в центре
  - Кнопки «Отмена» / «Готово»
  - Год (chevron-селектор)
