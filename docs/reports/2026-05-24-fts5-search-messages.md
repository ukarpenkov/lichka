## Принятые решения

- **External content FTS5** (`content=messages`) — не дублирует данные, индекс хранит только токенизированный `body`
- **`content_rowid=rowid`** — используется встроенный SQLite rowid (implicit integer PK), триггеры ссылаются на `new.rowid` / `old.rowid`
- **Префикс-匹配** (`"term"*`) — позволяет находить частичные совпадения (пользователь печатает «прив» → найдёт «привет»)
- **`highlight()`** — возвращает body с обрамлением совпадений тегами `<mark>`, готово для UI
- **`rank`** сортировка — FTS5 отдаёт наиболее релевантные первыми

## Известные ограничения

- FTS5 не поддерживает поиск по кириллице из коробки — нужен `unicode61` токенизатор или кастомный; дефолтный `unicode61` работает для базовых случаев
- Поиск только по полю `body` — если потребуется поиск по другим полям, нужно расширить виртуальную таблицу
- Нет пагинации результатов — при необходимости добавить `LIMIT` / `OFFSET`

## Тестирование

- Миграция применяется автоматически при вызове `runMigrations()` (триггеры и FTS-таблица создаются с `IF NOT EXISTS`)
- Для проверки: вставить сообщение в `messages`, затем вызвать `searchMessages('ключевое слово')`
