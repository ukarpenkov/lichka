# Исправление багов ChatForm

**Дата:** 2026-06-28
**Промпт/задача:** Исправить 3 бага: Sheet открывается под клавиатурой, кнопка Create не читается, тап по аватарке

## Что сделано

### 1. Sheet открывается под клавиатурой
- Заменил `BottomSheetModal` (@gorhom/bottom-sheet) на нативный `Modal` с `animationType="slide"` и `transparent`
- Лист теперь открывается сверху экрана, контент всегда виден даже при открытой клавиатуре
- Обёрнул содержимое в `KeyboardAvoidingView` (iOS `padding`, Android без изменений)
- Добавил кнопку закрытия (X) в шапке

### 2. Кнопка Create не читается
- **Корневая причина:** `Button.tsx` использовал `color: text` для текста, а `ChatForm` ставил `backgroundColor: text` — одинаковый цвет фона и надписи
- **Исправление:** `Button.tsx` теперь определяет `backgroundColor` из переданных стилей и инвертирует цвет текста — если фон = text, текст = background

### 3. Тап по аватарке
- Аватар-превью теперь кликабельно: нет аватара → галерея, есть emoji → emoji-пикер
- Добавлена функция `handleAvatarTap` с этой логикой
- **Баг с сохранением emoji:** в `handleSave` emoji не сохранялся в БД (`avatarPath = null` вместо строки emoji). Исправлено: `avatarPath = emojiAvatar`
- При редактировании чата корректно определяется тип аватара (emoji vs фото) по наличию `/` или `\` в пути

## Изменённые файлы
- `src/widgets/chat-form/ChatForm.tsx` — замена BottomSheetModal на Modal, тап по аватарке, фикс сохранения emoji
- `src/shared/ui/Button.tsx` — автоинверсия цвета текста при контрастном фоне

## Принятые решения
- Использован нативный `Modal` вместо `@gorhom/bottom-sheet` для предсказуемого поведения с клавиатурой
- Button компонент автоматически определяет нужный цвет текста — не требует ручной настройки от потребителей

## Известные ограничения
- Тестов на ChatForm и Button нет
- Предсуществующие ошибки TypeScript (AudioRecorderPlayer) не затронуты

## Тестирование
