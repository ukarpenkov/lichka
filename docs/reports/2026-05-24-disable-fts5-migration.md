# Отключение FTS5 миграции из-за отсутствия модуля в op-sqlite

**Дата:** 2026-05-24
**Промпт/задача:** Ошибка `no such module: fts5` при запуске приложения

## Что сделано
- Миграция 2 (FTS5 virtual table + триггеры) закомментирована в `src/shared/db/db.ts`
- Добавлен TODO для повторного включения после настройки op-sqlite с поддержкой FTS5

## Изменённые файлы
- `src/shared/db/db.ts` — миграция 2 закомментирована

## Принятые решения
- `op-sqlite` v16.1.0 поставляется без модуля FTS5 по умолчанию. Для включения нужен конфиг `op-sqlite.json` в корне проекта + пересборка нативных библиотек
- Функция `searchMessages` экспортируется, но нигде не вызывается — падения не будет
- FTS5 будет включён отдельной задачей, когда понадобится полнотекстовый поиск

## Известные ограничения
- Полнотекстовый поиск (`searchMessages`) нерабочий до включения FTS5
- У устройств, где миграция 2 уже была применена, таблица `messages_fts` останется в БД (не мешает)

## Тестирование
- Холодный запуск — миграции 1 и 3 применяются успешно, экран отображается
- Повторный запуск — миграции пропускаются (уже в schema_migrations)
