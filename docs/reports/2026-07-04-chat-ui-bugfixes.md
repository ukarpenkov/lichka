# Исправление багов интерфейса чата: кнопка вложений и переполнение изображений

**Дата:** 2026-07-04
**Промпт/задача:** Исправить два бага в интерфейсе чата — неправильное расположение кнопки вложений и выход изображения за границы bubble сообщения.

## Что сделано

### Bug #1 — Кнопка вложений внутри поля ввода
- Кнопка Paperclip перенесена из панели `actions` внутрь поля ввода текста
- `TextInput` и `IconButton(Paperclip)` обёрнуты в общий контейнер `inputWrapper` с `flexDirection: 'row'`
- Поле ввода теперь имеет `flex: 1`, а кнопка прикреплена справа
- Стилизация поля ввода (border, borderRadius) перенесена с TextInput на wrapper

### Bug #2 — Изображение выходит за границы bubble
- В `ImageMessage` добавлены отрицательные отступы (`marginHorizontal: -12`, `marginVertical: -8`) для компенсации padding bubble
- Изображение теперь заполняет bubble от края до края
- Добавлен `overflow: 'hidden'` в bubble для изображений, чтобы borderRadius обрезал углы
- Углы изображения скругляются согласно bubble: сверху всегда radius=16, снизу — только если нет подписи
- Подпись к изображению имеет собственные отступы (`paddingHorizontal: 12`, `paddingBottom: 8`)

## Изменённые файлы
- `src/widgets/message-composer/MessageComposer.tsx` — перенос Paperclip внутрь поля ввода
- `src/widgets/image-message/ImageMessage.tsx` — отрицательные отступы, правильный borderRadius, отступы для подписи
- `src/pages/chat-room/MessageBubble.tsx` — добавлен `overflow: 'hidden'` для image-сообщений
- `docs/bugs/attachment-button-and-image-overflow.md` — описание багов

## Принятые решения
- Для Bug #2 использован подход с отрицательными отступами вместо условного удаления padding у bubble. Это менее инвазивно и позволяет ImageMessage быть самодостаточным компонентом
- `overflow: 'hidden'` добавлен условно только для изображений, чтобы не влиять на остальные типы сообщений
