# Исправление отображения выбора эмодзи при создании/редактировании чата

**Дата:** 2026-07-01
**Промпт/задача:** Не видно выбора эмодзи для чата если нажать на эмодзи при редактировании чата или создании нового чата — только внизу экрана появляется крестик, но самих эмодзи нет

## Что сделано

- Добавлено условное свойство `flex: 1` на контейнер `sheetContent` при открытой панели эмодзи (`showEmojiPicker && styles.sheetContentExpanded`)
- Добавлен новый стиль `sheetContentExpanded: { flex: 1, paddingHorizontal: 0 }` — убран боковой padding, чтобы эмодзи не обрезались по краям
- Увеличены размеры ячеек эмодзи: ширина/высота 48→52px, margin 4→5, fontSize 28→32, borderRadius 8→10

## Изменённые файлы

- `src/widgets/chat-form/ChatForm.tsx` — строка 169: добавлен условный стиль; строка 251-254: обновлён `sheetContentExpanded`
- `src/widgets/chat-form/EmojiGrid.tsx` — строки 61-70: увеличены размеры ячеек и шрифта эмодзи

## Принятые решения

Проблема: контейнер `sheetContent` не имел `flex: 1` или явной высоты. React Native определяет высоту по содержимому, а дочерний `EmojiGrid` с `flex: 1` схлопывался в 0px. Отображалась только шапка с крестиком.

Дополнительно: при `paddingHorizontal: 20` на узких экранах (360px) ячейки эмодзи не помещались в 6 колонок и обрезались. Увеличен размер ячеек для лучшей читаемости и таргетинга.

Решение: условно добавлять `flex: 1` и `paddingHorizontal: 0` на `sheetContent` когда активен пикер эмодзи. Без условия `flex: 1` растянул бы форму создания/редактирования чата на 90% экрана, что создало бы лишнее пустое пространство.
