# Центрирование времени и локальные часы в DateTimePicker

**Дата:** 2026-05-31
**Промпт/задача:** Часы и минуты не по центру круга, часы «вылетают» за кольцо; при открытии должны показываться текущие локальные часы и минуты.

## Что сделано
- Состояние даты/времени переведено с UTC (`getUTC*`, `Date.UTC`) на локальное (`get*`, `new Date(y, m, d, h, min)`), согласовано с `formatScheduledAt` и `new Date()` при открытии.
- `TimeScroller` вынесен из `GestureDetector` колец — FlatList больше не внутри жеста вращения, вёрстка не ломается.
- Добавлена обрезка `timeScrollerClip` по радиусу центра (`GEO.centerRadius`), `zIndex` для слоя времени.
- В `TimeScroller`: синхронизация прокрутки при смене `hour`/`minute` извне (`useEffect`), `flexGrow: 0` / `overflow: 'hidden'` у колонок.

## Изменённые файлы
- `src/widgets/datetime-picker/DateTimePicker.tsx` — локальное время, структура центра.
- `src/widgets/datetime-picker/TimeScroller.tsx` — синхронизация скролла и стили списков.

## Принятые решения
- Хранение по-прежнему через `date.toISOString()` на стороне вызывающего кода; пикер отдаёт локальный `Date`, ISO корректно отражает момент в TZ устройства.
- Жест колец не оборачивает скролл времени — центр остаётся для `FlatList`, кольца — отдельный слой.

## Известные ограничения
- Ручная проверка на устройстве в сессии не выполнялась.

## Тестирование
- Открыть picker → время совпадает с системным; часы/минуты по центру круга, без смещения влево.
