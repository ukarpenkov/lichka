# Day bezel: чувствительность ×0.5 и визуальный шаг ×1.5

**Дата:** 2026-07-22
**Промпт/задача:** Снизить чувствительность перелистывания дат на безеле в 2 раза (только дни); визуально увеличить расстояние между датами в 1.5 раза.

## Что сделано

- `DAY_ROTATION_GAIN`: `0.62` → `0.31` (в 2 раза меньше отклик кольца на движение пальца)
- `DAY_SPACING = 1.5` — угловой шаг дней `(360 / count) * 1.5`
- `Bezel` / `BezelLabel` принимают опциональный `step`, чтобы визуал совпадал с жестом
- Позиционирование лейблов через shortest path в пространстве индексов — без наложений, когда `count * step > 360`
- Месяцы без изменений; `DAY_STEP_THRESHOLD = 0.62` сохранён

## Изменённые файлы

- `src/widgets/datetime-picker/DateTimePicker.tsx` — gain, spacing, `dayStep`, проп `step` у day Bezel
- `src/widgets/datetime-picker/Bezel.tsx` — проброс `step`
- `src/widgets/datetime-picker/BezelLabel.tsx` — index-based позиция/fade + `step`

## Принятые решения

- Визуальный шаг и жест используют один и тот же `dayStep(dayCount)`, чтобы выбранный день оставался наверху
- Порог коммита дня не трогали: при меньшем gain он и так требует большего хода пальца

## Известные ограничения

- Gain/spacing снова подобраны «на глаз»; при необходимости можно подкрутить `DAY_ROTATION_GAIN` / `DAY_SPACING`

## Тестирование

- Линт изменённых файлов — без замечаний по сути правок
- Ручная проверка: кольцо дней — шаг на 1 день без перескока; соседи визуально дальше; месяцы как раньше
