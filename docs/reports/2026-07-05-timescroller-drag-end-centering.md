# Исправление центрирования TimeScroller при медленной прокрутке

**Дата:** 2026-07-05
**Промпт/задача:** Баг — после прокрутки селектора часов/минут выбранное значение не доводится до центральной позиции

## Что сделано

- Добавлены обработчики `onScrollEndDrag` для FlatList часов и минут
- При нулевой скорости прокрутки (нет инерции) вызывается логика центрирования, идентичная `onMomentumScrollEnd`
- При наличии инерции обработчик не вмешивается — финальный снап происходит через `onMomentumScrollEnd`

## Изменённые файлы

- `src/widgets/datetime-picker/TimeScroller.tsx` — добавлены `handleHourDragEnd`, `handleMinuteDragEnd` и пропы `onScrollEndDrag`
- `docs/bugs/timescroller-drag-end-centering.md` — баг-репорт

## Принятые решения

- Порог скорости `0.01` — ниже считается «нет инерции», выше — «есть инерцию, подождём onMomentumScrollEnd»
- Предыдущий фикс удалил `onScrollEndDrag` целиком, чтобы не ломать инерцию; теперь он добавлен обратно с защитой по скорости

## Известные ограничения

- Нет автоматизированных тестов для UI-компонентов с прокруткой

## Тестирование

- Ручное: медленная прокрутка → значение выравнивается по центру
- Ручное: быстрый свайп → инерция работает, центрирование корректное
