# Переход из «Запланировано»: открыть чат без будущих сообщений

**Дата:** 2026-07-20
**Промпт/задача:** Из «Запланировано» не показывать будущее сообщение в ленте — просто открывать чат; для periodic подсвечивать последнее сработавшее. Ранее (см. `2026-07-20-scheduled-delete-and-scroll-highlight.md`) делали обратное — это было ошибкой.

## Что сделано
- Тап по reminder/alarm из «Запланировано» открывает чат **без** `messageId` — без скролла, подсветки и inject будущих пузырьков.
- Тап по periodic передаёт `messageId` шаблона: чат скроллит к display-сообщению `periodic:<id>` (последнее срабатывание) и кратко подсвечивает его.
- Убран временный inject будущих reminder/alarm/не-сработавших periodic в `ChatRoomScreen.loadData`.

## Изменённые файлы
- `src/pages/scheduled/ScheduledScreen.tsx` — `handlePress` различает periodic и остальные типы
- `src/pages/chat-room/ChatRoomScreen.tsx` — удалён inject целевого сообщения по `messageId`

## Принятые решения
- Скролл/подсветка из «Запланировано» только для periodic (последний display-экземпляр уже в ленте через `getPeriodicDisplayMessages`).
- Путь уведомлений / поиска с `messageId` не менялся — сработавшие сообщения уже видимы в ленте.
- Если periodic ещё ни разу не сработал (`fires === 0`), display-сообщения нет: чат просто открывается без подсветки.

## Известные ограничения
- Коммит не создавался — в правилах пользователя коммит только по явному запросу.

## Тестирование
- Ручная проверка: тап по будущему reminder/alarm → чат без «будущего» пузырька.
- Ручная проверка: тап по periodic после хотя бы одного срабатывания → скролл + подсветка последнего.
