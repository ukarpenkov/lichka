# Исправление: последнее сообщение перекрывается composer при клавиатуре

**Дата:** 2026-07-08
**Промпт/задача:** Описать баг в docs/bugs и исправить перекрытие последнего сообщения блоком ввода при активной клавиатуре.

## Что сделано
- Создан `docs/bugs/chat-last-message-hidden-behind-composer.md` с описанием бага, шагами воспроизведения и причиной.
- Добавлена константа `CHAT_LIST_KEYBOARD_BOTTOM_INSET` (28 px = `KEYBOARD_COMPOSER_GAP + 12`) в `shared/lib/keyboard.ts`.
- В `ChatRoomScreen` список сообщений получает анимированный `paddingBottom` при открытой клавиатуре, чтобы компенсировать `translateY` composer и оставить визуальный зазор.

## Изменённые файлы
- `docs/bugs/chat-last-message-hidden-behind-composer.md` — описание бага
- `src/shared/lib/keyboard.ts` — константа `CHAT_LIST_KEYBOARD_BOTTOM_INSET`
- `src/shared/lib/index.ts` — экспорт константы
- `src/pages/chat-room/ChatRoomScreen.tsx` — анимированный отступ списка при клавиатуре

## Принятые решения
- Отступ списка привязан к `KEYBOARD_COMPOSER_GAP`, чтобы при изменении зазора composer автоматически обновлялась компенсация.
- Дополнительные 12 px — визуальный зазор между bubble и полем ввода (по запросу «чуть больше отступ»).

## Известные ограничения
- Фиксированное значение отступа; при существенном изменении высоты composer (например, превью изображения) может потребоваться доработка.

## Тестирование
- Ручная проверка: открыть чат → активировать клавиатуру → последнее сообщение полностью видно над composer.
- `npx tsc --noEmit` — ошибок в изменённых файлах нет.
