# Исправление зазора под инпутом после повторного входа в чат (вторая итерация)

**Дата:** 2026-07-05
**Промпт/задача:** Баг-репорт: при навигации «назад» из чата с открытой клавиатурой и повторном входе, блок ввода сообщения остаётся приподнятым, образуя пустое пространство снизу.

## Что сделано

- **`src/shared/lib/keyboard.ts`** — полностью убран `useAnimatedKeyboard` и блок `runOnUI`, синхронизировавший состояние с Reanimated. JS‑лисенеры `keyboardDidShow`/`keyboardDidHide` теперь единственный источник правды. В cleanup добавлен `runOnUI(() => { kbHeight.value = 0 })` для принудительного сброса.
- **`src/pages/chat-room/ChatRoomScreen.tsx`** — добавлен `useEffect(() => { Keyboard.dismiss() }, [])` на mount как страховка.

## Принятые решения

- Первая попытка исправления (замена `useDerivedValue` на `useSharedValue`) была недостаточной: `runOnUI`-синхронизация с `useAnimatedKeyboard()` реинтродьюсила stale‑значение OPEN при повторном mount на Android с `adjustNothing`.
- `useAnimatedKeyboard()` полностью удалён из хука — на Android с `adjustNothing` + `react-native-screens` его состояние ненадёжно при перемонтировании native view.
- Случай «клавиатура уже открыта при mount» (deep link) обрабатывается через `Keyboard.dismiss()` на mount в `ChatRoomScreen` — пользователь может тапнуть по инпуту, чтобы открыть клавиатуру заново.

## Известные ограничения

- При deep link в чат во время ввода текста клавиатура будет принудительно закрыта. Пользователю потребуется тапнуть по инпуту для повторного открытия. Это приемлемый компромисс.
