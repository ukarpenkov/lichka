# Изменение порядка сообщений в чате

**Дата:** 2026-06-30
**Промпт/задача:** Новые сообщения чата должны добавляться вниз, а не вверх

## Что сделано

- Убрано свойство `inverted` из FlatList в `ChatRoomScreen.tsx` — ранее оно инвертировало порядок, из-за чего новые сообщения отображались сверху
- Добавлен `useEffect` с `scrollToEnd` для автоматической прокрутки к последнему сообщению при загрузке чата и после отправки
- Введён ref `scrollToMessageId` для предотвращения конфликта автопрокрутки с навигацией к конкретному сообщению (поиск, deep link)

## Изменённые файлы

- `src/pages/chat-room/ChatRoomScreen.tsx` — удалён `inverted`, добавлена автопрокрутка вниз, добавлен флаг навигации

## Принятые решения

- Порядок данных в SQL остаётся `ASC` (старые первые) — без `inverted` это даёт нужный визуальный порядок
- Автопрокрутка через `scrollToEnd` с задержкой 100ms для стабильности рендера
- Флаг `scrollToMessageId` устанавливается при навигации к сообщению и блокирует автопрокрутку

## Известные ограничения

- При открытии клавиатуры最新ые сообщения могут перекрываться композером — это было и ранее (симметричная проблема)
- Sticky date header продолжает работать корректно — `handleViewableItemsChanged` возвращает первый видимый date-separator сверху

## Тестирование

- Ручная проверка: новые сообщения отображаются внизу
- Проверка навигации к сообщению через поиск
- Проверка автопрокрутки после отправки нового сообщения
