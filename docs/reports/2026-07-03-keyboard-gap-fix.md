# Фикс зазора между блоком ввода и клавиатурой

**Дата:** 2026-07-03
**Промпт/задача:** Убрать пустой зазор (~20–40px) между MessageComposer и верхней границей клавиатуры при открытии чата

## Что сделано
- В `MessageComposer.tsx` добавлен `useAnimatedKeyboard()` для отслеживания видимости клавиатуры
- Создан `containerAnimatedStyle`, который устанавливает `paddingBottom: 0` при открытой клавиатуре и `paddingBottom: 12` при скрытой (дефолтный отступ снизу)
- Стиль применён к обоим состояниям контейнера: обычному и состоянию записи голоса

## Изменённые файлы
- `src/widgets/message-composer/MessageComposer.tsx` — добавлен `useAnimatedKeyboard`, анимированный стиль `containerAnimatedStyle`, применён к двум `Animated.View`

## Принятые решения
- **Причина бага:** `MessageComposer.container` имеет хардкодный `paddingBottom: 12`, который создавал видимый зазор между иконками и клавиатурой на обеих платформах
- **Решение:** динамическое переключение `paddingBottom` через `useAnimatedKeyboard().height.value` — когда клавиатура открыта (height > 0), отступ 0, иначе 12
- Платформа не детектится — `useAnimatedKeyboard` из reanimated v3 работает на обеих платформах, а на iOS система сама ресайзит view через `adjustResize`
- `ChatRoomScreen.tsx` не трогался — его логика (`adjustNothing` + анимированный `paddingBottom`) работает корректно, проблема была исключительно во внутреннем padding'е композера

## Известные ограничения
- На Android с `adjustNothing` клавиатура оверлеится поверх окна без ресайза — `useAnimatedKeyboard` даёт корректную высоту только если клавиатура уже «прикреплена» к окну

## Тестирование
- Открыть чат → тапнуть поле ввода → проверить, что MessageComposer прижат к клавиатуре без зазора
- Закрыть клавиатуру → проверить, что отступ 12px снизу возвращается
- Проверить состояние записи голоса (долгое нажатие на микрофон) — панель записи тоже должна прижиматься к клавиатуре
