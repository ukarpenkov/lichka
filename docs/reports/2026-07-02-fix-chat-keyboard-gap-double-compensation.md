# Исправление зазора клавиатуры из-за двойной компенсации

**Дата:** 2026-07-02
**Промпт/задача:** Оформленный баг-репорт: лишний отступ между панелью ввода сообщения и клавиатурой в экране чата

## Что сделано
- Удалён `useAnimatedKeyboard` из `ChatRoomScreen`
- Удалён `keyboardAreaStyle` — анимированный стиль с `paddingBottom` на Android
- Удалены неиспользуемые импорты (`Platform`, `useAnimatedKeyboard`, `interpolate`, `Extrapolation`)
- `Animated.View`-обёртка вокруг чата заменена на обычный `View`

## Изменённые файлы
- `src/pages/chat-room/ChatRoomScreen.tsx` — удалена избыточная клавиатурная логика

## Принятые решения
**Проблема:** Двойная компенсация высоты клавиатуры:
- `adjustResize` в AndroidManifest уже уменьшает окно на высоту клавиатуры
- `paddingBottom` из `useAnimatedKeyboard` добавлял вторую компенсацию
- Итог: контент сдвигался слишком высоко, между `MessageComposer` и клавиатурой появлялся зазор

**Решение:** Полностью полагаться на нативную обработку клавиатуры через `adjustResize` на Android и на системную обработку на iOS. Убрать ручную компенсацию.
