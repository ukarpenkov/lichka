# Дизайн экрана будильника на основе alarm-flat.html

**Дата:** 2026-07-04
**Промпт/задача:** Создать экран будильника по прототипу `design/layout/alarm-flat.html` без CDN, без крестика у кнопки «Отключить», без надписи «Звонок», без переключателей тем сверху, с сохранением анимации и качественной иконкой.

## Что сделано

- Создан экран будильника `src/pages/alarm/AlarmScreen.tsx` с анимациями на `react-native-reanimated`
- Pulse-анимация: два кольца расх‌одятся и затухают с задержкой 600ms (как в прототипе)
- Shake-анимация: иконка будильника покачивается (±4°)
- Иконка будильника: используется существующий `AlarmClockIcon` (SVG, strokeWidth=2) в размере 72px
- Кнопка «Отключить» — красный текст без иконки-крестика
- Кнопка «Отложить · 5 мин» — серый текст с иконкой `Clock` (lucide)
- Разделитель между кнопками
- Home indicator внизу
- Экран добавлен как fullScreenModal через корневой Stack-навигатор

## Изменённые файлы

- `src/pages/alarm/AlarmScreen.tsx` — экран будильника с анимациями
- `src/pages/alarm/index.ts` — barrel export
- `src/pages/index.ts` — добавлен экспорт `AlarmScreen`
- `src/app/types.ts` — добавлен `RootStackParamList` с маршрутами `Main` и `Alarm`
- `src/app/AppNavigator.tsx` — корневой `RootStack`, табы вынесены в `MainTabs`, `AlarmScreen` как fullScreenModal

## Принятые решения

- **Корневой Stack**: табы обёрнуты в `RootStack.Navigator`, экран будильника — модальный с анимацией `fade` и `fullScreenModal`
- **Цвета**: используются theme-цвета с прозрачностью (`text + '14'` для границ, `text + '88'` для иконки и т.д.), акцентный красный `#FF6B6B` для кнопки отключения
- **Иконка**: существующий `AlarmClockIcon` из `shared/ui` — переиспользование вместо дублирования
- **Адаптивный размер**: иконка масштабируется под ширину экрана (`screenWidth * 0.38`, максимум 148px)
- **Текст**: жёстко на русском (как в прототипе)

## Известные ограничения

- Текст кнопок жёстко задан на русском, без локализации
- Интеграция с нативным модулем (запуск экрана при срабатывании будильника) — отдельная задача
- `handleDismiss` и `handleSnooze` сейчас просто закрывают экран через `navigation.goBack()`

## Тестирование

- TypeScript: без новых ошибок (единственная оставшаяся — `absoluteFillObject` в `AlertDialog.tsx`, предсуществующая)
- ESLint: без ошибок, 3 предупреждения `no-unstable-nested-components` в `AppNavigator.tsx` — предсуществующие (inline tabBarIcon)
