# Исправление инерции и центрирования TimeScroller

**Дата:** 2026-07-05
**Промпт/задача:** Исправить два бага в TimeScroller: отсутствие инерции при свайпе и неточное центрирование выбранного значения

## Что сделано

- Удалён `disableIntervalMomentum` — список теперь сохраняет инерцию после свайпа
- Исправлен расчёт offset в `scrollToDataIndex`: `(dataIdx - 1) * ITEM_HEIGHT` вместо `dataIdx * ITEM_HEIGHT` — элемент помещается в центральную ячейку (highlight), а не верхнюю
- Аналогично исправлены `recenterHours` и `recenterMinutes`
- Удалён `onScrollEndDrag` — финальный снап только через `onMomentumScrollEnd`, после завершения инерции

## Изменённые файлы

- `src/widgets/datetime-picker/TimeScroller.tsx` — три исправления
- `docs/bugs/timescroller-scroll-inertia-centering.md` — описание бага

## Принятые решения

- `onScrollEndDrag` удалён полностью вместо условной обработки — проще и надёжнее, снап только после полной остановки
- Offset `-1 ITEM_HEIGHT` компенсирует то, что highlight расположен на втором из трёх видимых элементов

## Известые ограничения

- Тестов на TimeScroller нет

## Тестирование

- Ручное: проверить инерцию при быстром свайпе, точность центрирования после остановки
