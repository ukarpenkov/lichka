# DatePicker: скрытый свайп года / месяца / дня

**Дата:** 2026-07-20
**Промпт/задача:** Скрытая фича — горизонтальный свайп листает год, месяц и день; при смене дня/месяца безель синхронизируется.

## Что сделано
- В шапке DateTimePicker три зоны свайпа:
  - **год** — вокруг `YearPicker`
  - **месяц** — подпись полного имени месяца
  - **день** — крупное число дня
- Направление: свайп влево → следующее значение (+1), вправо → предыдущее (−1)
- Пороги: смещение ≥ 36 px или velocity ≥ 420
- Год ограничен `MIN_YEAR`/`MAX_YEAR` (2020–2035)
- Месяц зациклен 0…11; день зациклен в пределах длины текущего месяца
- При свайпе месяца/дня `monthRotation` / `dayRotation` обновляются через `withSpring` — безель следует за значением
- Haptic при успешном шаге (если включён в настройках)

## Изменённые файлы
- `src/widgets/datetime-picker/DateTimePicker.tsx` — pan-жесты, step-хелперы, sync безеля
- `src/widgets/datetime-picker/YearPicker.tsx` — экспорт `MIN_YEAR` / `MAX_YEAR`

## Принятые решения
- Зоны разделены по подписям в шапке (скрыто, без UI-хинтов) — не пересекаются с кольцевым gesture на безеле
- Без каскада (день→месяц→год): каждая зона меняет только своё поле
- Стрелки года и long-press на год сохранены; pan активируется только после горизонтального offset

## Известные ограничения
- На границах года (2020/2035) свайп «мимо» диапазона игнорируется
- Визуального follow-finger translate нет — только смена значения + spring безеля

## Тестирование
- Линт изменённых файлов — без ошибок
- Вручную: свайп по году / месяцу / дню → значение и кольца совпадают; кольцевой drag безеля не сломан
