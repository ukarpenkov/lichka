# Разблокировка панели типов сообщений при выбранном изображении

**Дата:** 2026-07-05
**Промпт/задача:** При выборе картинки через скрепку панель типов сообщений (напоминание, будильник, периодическое) должна оставаться доступной. Блокироваться должен только микрофон (диктофон), так как голосовое + картинка несовместимы.

## Что сделано

- Убран условный рендеринг `{!imagePreview && (...)}` вокруг кнопок Repeat, AlarmClock, Bell — теперь они видны всегда
- Кнопка микрофона по-прежнему скрывается при наличии `imagePreview`
- `sendMessage` переписан: теперь он асинхронный и обрабатывает изображения внутри себя — сохраняет в storage, формирует payload, подставляет фолбэк-текст при отсутствии текста
- Функция `sendImageMessage` удалена — её логика поглощена `sendMessage`
- `handleRemoveImage` больше не очищает поле ввода текста при удалении превью картинки
- `handleSend` упрощён — всегда вызывает `sendMessage('simple')`, который сам решает, какой тип сообщения создать (`'image'` или `'simple'`)
- Условие `disabled` для кнопок Repeat/AlarmClock/Bell изменено с `!body.trim()` на `!imagePreview && !body.trim()` — кнопки активны, когда есть выбранное изображение (даже без текста)

## Изменённые файлы

- `src/widgets/message-composer/MessageComposer.tsx` — основная логика и UI
- `src/widgets/message-composer/__tests__/MessageComposer.test.tsx` — обновлено название теста

## Принятые решения

- **Тип сообщения с картинкой и scheduled-типом:** если выбрано изображение и тип напоминание/будильник/периодическое, то тип в БД остаётся `'reminder'`/`'alarm'`/`'periodic'`, а данные изображения кладутся в `payload`. Это позволяет `ImageMessage` корректно рендерить такие сообщения (он смотрит только на `payload`).
- **Тип `'simple'` + картинка:** автоматически преобразуется в `'image'` (как и было раньше).
- **Тело сообщения:** если есть текст пользователя — используется он. Если текста нет, но есть картинка — используется `t.imageMessage(w, h)` (`[image:800x600]`). Если нет ни текста, ни картинки — отправка не происходит.
- **Микрофон:** скрывается при наличии изображения, так как голосовое сообщение с картинкой не имеет смысла.

## Известные ограничения

- Выбор нескольких изображений пока не поддерживается (пользователь выбирает одно фото через `launchImageLibrary`)
- Нет индикатора того, что к scheduled-сообщению прикреплено изображение (в списке чатов scheduled-сообщения выглядят одинаково с картинкой и без)
