# Startup cleanup orphan медиафайлов

**Дата:** 2026-05-26
**Промпт/задача:** 10.2 Реализовать удаление сообщений и чатов — часть «Startup cleanup»

## Что сделано
- Создан модуль `cleanupOrphanMedia()` в `src/shared/lib/cleanupMedia.ts`
  - Обходит поддиректории `media/` (avatars/, voice/)
  - Собирает все ссылки на медиафайлы из БД: `chats.avatar_path` и `messages.payload` → `uri`
  - Удаляет файлы, на которые нет ссылок в БД
  - Возвращает список удалённых файлов
- Функция вызывается при старте приложения в `App.tsx` (fire-and-forget)

## Изменённые файлы
- `src/shared/lib/cleanupMedia.ts` — новый модуль cleanup
- `src/shared/lib/index.ts` — добавлен экспорт `cleanupOrphanMedia`
- `App.tsx` — вызов `cleanupOrphanMedia()` на старте

## Принятые решения
- Cleanup запускается асинхронно, не блокирует запуск приложения
- Ошибки удаления отдельных файлов игнорируются (graceful degradation)
- Модуль размещён в `shared/lib/` — кросс-сущностная утилита

## Известные ограничения
- Файлы в корне `media/` (не в поддиректориях) не обрабатываются — текущая структура не предполагает таких файлов
