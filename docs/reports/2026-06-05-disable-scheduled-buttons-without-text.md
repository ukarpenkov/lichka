# Отключение кнопок reminder/alarm/periodic без текста

**Дата:** 2026-06-05
**Промпт/задача:** Сделать кнопки reminder, alarm, periodic disabled когда нет текста в поле ввода (как кнопку отправки)

## Что сделано

- Кнопки **Bell** (reminder), **AlarmClock** (alarm), **Repeat** (periodic) теперь disabled когда `body` пустой
- Визуальная индикация: `disabled` prop + opacity 0.4 от IconButton (цвет статичный, без тернарников)
- Убран fallback `t.noText` в `sendMessage` — текст обязателен для всех типов сообщений
- Добавлена проверка `if (!textBody) return;` в `sendMessage` для всех типов

## Изменённые файлы

- `src/widgets/message-composer/MessageComposer.tsx` — кнопки Bell, AlarmClock, Repeat получили `disabled={!body.trim()}` и условный цвет
- `src/shared/config/locale.ts` — удалена неиспользуемая строка `noText` из типа и обоих языков

## Принятые решения

- **Все три типа** (reminder, alarm, periodic) требуют текст — это «сообщение себе», а не просто будильник
- Кнопка **Mic** осталась без изменений — запись голоса не зависит от текста
- Кнопка **Send** уже имела эту логику, остальные приведены к единому поведению

## Известные ограничения

- Нет

## Тестирование

- Визуально: кнопки серые при пустом поле, становятся активными при вводе текста
- При очистке текста кнопки снова disabled
