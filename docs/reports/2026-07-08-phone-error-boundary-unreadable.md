# Ошибка при открытии приложения на телефоне (полоски вместо текста)

**Дата:** 2026-07-08
**Промпт/задача:** На телефон устанавливается приложение, при открытии — экран с «!» и непонятным текстом (полоски). Дополнение: как пересобрать.

## Что сделано

- Диагностирован экран: это не «битый шрифт», а кастомный `ErrorBoundary`, который **намеренно** рисовал три прямоугольника вместо текста ошибки.
- `ErrorBoundary` теперь показывает читаемое сообщение (`error.message`) и логирует ошибку в `componentDidCatch`.
- В `AppInitProvider` каждый шаг инициализации обёрнут в try/catch с префиксом (`Database migration failed`, `Seed default chat failed`, `Notification channels failed`).
- В `notificationChannels.ts` добавлена проверка, что `NotificationModule` слинкован (иначе понятная ошибка вместо `Cannot read property of undefined`).
- Добавлена инструкция по пересборке для установки на телефон (см. ниже / ответ в чате).

## Изменённые файлы

- `src/app/ErrorBoundary.tsx` — показ текста ошибки вместо skeleton-полосок
- `src/app/AppInitProvider.tsx` — пошаговая диагностика init
- `src/shared/lib/notificationChannels.ts` — guard на отсутствие native module
- `docs/reports/2026-07-08-phone-error-boundary-unreadable.md` — отчёт

## Принятые решения

- Показывать текст ошибки всегда (не только в `__DEV__`), чтобы на release-сборке на телефоне было видно причину без `adb logcat`.
- Stack trace — только в `__DEV__`.
- Не чинить конкретный runtime-баг вслепую: на подключённом эмуляторе текущая сборка стартует; на телефоне нужна пересборка с новым ErrorBoundary, чтобы увидеть точный текст.

## Наиболее вероятные причины (по коду)

1. JS-ошибка при старте (`runMigrations` / `seedDefaultChat` / `registerNotificationChannels`) → ErrorBoundary.
2. Ошибка первого рендера после init (навигация / native modules) → тот же ErrorBoundary.
3. Ранее (сегодня) уже чинился баг `Another navigator is already registered` в `SwipeablePager` — если на телефон попала сборка **до** этого фикса, экран ошибки ожидаем.

## Известные ограничения

- Точная причина на физическом телефоне пользователя ещё не подтверждена логами (телефон не был в `adb devices`, только эмулятор).
- Нужна переустановка APK/сборки с этими правками, чтобы увидеть текст ошибки на экране.

## Тестирование

- Эмулятор `emulator-5554`: `com.lichka` запускается, в logcat нет `ErrorBoundary` / `AppInit` / JS TypeError при cold start после `pm clear`.
- Линтер по изменённым файлам — без замечаний.
- Следующий шаг для пользователя: пересобрать и поставить на телефон; если ошибка останется — прислать текст с экрана.
