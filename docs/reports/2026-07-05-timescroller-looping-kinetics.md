# Доработка TimeScroller: зацикливание + reduce motion

**Дата:** 2026-07-05
**Промпт/задача:** Добавить зацикливание часов (23 → 00) и минут (59 → 00), поддержку reduce motion для TimeScroller.

## Критический разбор предложенного кода

Предложенная реализация (из промпта) имела следующие проблемы:

1. **Смена направления скролла** с вертикального на горизонтальный — ломает существующий дизайн (две вертикальные колонки с разделителем `:`) и требует переписывания интеграции в `DateTimePicker.tsx`.
2. **Нарушение правил React Hooks** — `useAnimatedStyle` вызывается в цикле внутри `render`, что запрещено правилами хуков.
3. **Дублирующиеся определения жеста** — объявлены `gesture`, затем `panGesture` с конфликтующей логикой.
4. **Удаление тактильного отклика** — пропадает проп `onTick` для haptic feedback.
5. **Избыточная сложность** — 300+ элементов в ленте (5 копий × 60 минут), риск просадок производительности.
6. **Несовместимость API** — новые пропсы (`value`/`max`/`onChange`) несовместимы с текущим `DateTimePicker`.

## Что сделано

Реализована собственная версия фичи, сохраняющая существующую архитектуру:

### 1. Зацикливание через дублирование данных

- Массивы `hours` (0..23) и `minutes` (0..59) дублируются `COPIES = 3` раза.
- Стартовая позиция — в средней копии (`dataIdxForHour/Minate`).
- При скролле индекс нормализуется: `((idx % COUNT) + COUNT) % COUNT` → реальное значение 0..23 или 0..59.
- **Recenter**: после завершения скролла указатель мгновенно (без анимации) возвращается в среднюю копию — пользователь этого не замечает, визуально значение не меняется.

### 2. Reduce Motion

- Подписка на `AccessibilityInfo.isReduceMotionEnabled()` + `reduceMotionChanged`.
- При включённой настройке все анимированные скроллы заменяются на мгновенные (`animated: false`).

### 3. Сохранение существующего API

- Протокол пропсов не изменился — `hour`, `minute`, `onHourChange`, `onMinuteChange`, `onTick`.
- `DateTimePicker.tsx` не требует изменений.

### 4. Кинетика

- Оставлено существующее поведение FlatList: `decelerationRate="fast"` + `snapToInterval` + `disableIntervalMomentum`.
- Это даёт достаточный кинетический отклик без перехода на `withDecay` + Reanimated, что упростило бы код ценой полной смены жестовой модели.

## Изменённые файлы

- `src/widgets/datetime-picker/TimeScroller.tsx` — полная переработка логики данных и скролла.

## Принятые решения

| Вопрос | Решение | Обоснование |
|--------|---------|-------------|
| Горизонтальный скролл (из совета) | Оставлен вертикальный двухколоночный FlatList | Не ломать существующий UX и интеграцию с DateTimePicker |
| Количество копий | 3 (вместо предложенных 5) | Достаточно для практического использования, меньше элементов в DOM |
| withDecay (из совета) | Не применён | FlatList уже даёт хорошую кинетику через `decelerationRate`; переход на Gesture Handler потребовал бы полной переделки |
| Null-паддинг | Убран | При зацикливании отступы не нужны — recenter обеспечивает «бесконечность» без визуальных разрывов |
| Интерфейс пропсов | Сохранён старый (`hour`/`minute`/`onHourChange`/`onMinuteChange`) | Обратная совместимость с DateTimePicker |

## Известные ограничения

- Кинетика ограничена возможностями FlatList — нет кастомного `withDecay`. При сильном свайпе скролл останавливается быстрее, чем на iOS-пикере времени.
- При нажатии (Pressable) значение мгновенно меняется с анимацией скролла к средней копии — может быть заметна лёгкая «перемотка», если нажатое значение далеко от текущей позиции в данных.
