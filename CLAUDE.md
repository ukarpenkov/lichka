# Lichka — Claude Code Rules

## FSD Architecture

Проект следует **Feature-Sliced Design**.

### Слои (сверху вниз)

| Слой | Назначение |
|------|------------|
| `app` | Провайдеры, инициализация, навигация, темы |
| `pages` | Страницы/экраны |
| `widgets` | Самостоятельные UI-блоки из фич |
| `features` | Пользовательские действия, бизнес-логика |
| `entities` | Бизнес-сущности (User, Note, Category) |
| `shared` | UI-kit, утилиты, api-client |

### Обязательные правила

- **Зависимости только вниз:** app → pages → widgets → features → entities → shared
- **Без циклов** между слайсами одного уровня
- **Public API:** экспорт только через `index.ts` слайса
- **Один модуль — одна задача:** низкая связанность, высокая сцепленность

### При добавлении кода

1. Определи слой и слайс по назначению, не по типу файла
2. Не импортируй из верхних слоёв
3. Новые слайсы создавай с `index.ts` как единственной точкой экспорта

## Коммиты

- **Каждое изменение кода сопровождается коммитом** — без накопления незафиксированных правок
- Коммиты **атомарные**: один коммит = одно логическое изменение
- Формат **Conventional Commits**: `<type>(<scope>): <description>`

### Типы

| Тип | Когда |
|-----|-------|
| `feat` | Новая функциональность |
| `fix` | Исправление бага |
| `docs` | Документация |
| `style` | Форматирование |
| `refactor` | Рефакторинг без смены поведения |
| `test` | Тесты |
| `chore` | Конфиг, зависимости |

### Примеры

```
feat(notes): add voice note recording
fix(sync): resolve Google Drive auth timeout
docs: add FSD architecture rules
test(entities/note): add unit tests for Note model
```

### Порядок работы

1. Изменить код
2. Написать/обновить отчёт в `docs/reports/`
3. Создать коммит с корректным типом и scope

---

## Отчёты (docs/reports)

Каждый промпт и каждое изменение кода **обязаны** сопровождаться отчётом в `docs/reports/`.

### Когда создавать

- После любого изменения кода (создание, правка, удаление файлов)
- После завершения задачи по промпту пользователя, даже если код не менялся (анализ, ревью, планирование)

### Имя файла

```
docs/reports/YYYY-MM-DD-<краткое-описание-задачи>.md
```

### Шаблон отчёта

```markdown
# <Название задачи>

**Дата:** YYYY-MM-DD
**Промпт/задача:** <краткое описание запроса пользователя>

## Что сделано
- ...

## Изменённые файлы
- `path/to/file.ts` — ...

## Принятые решения
- ...

## Известные ограничения
- ...

## Тестирование
- ...
```

### Правила

- Отчёт создаётся или обновляется **в конце сессии работы** над задачей
- Один промпт = один отчёт; при продолжении задачи — обновляй существующий отчёт
- Отчёт пишется на русском языке
- Не дублируй содержимое в чат — ключевые итоги кратко, детали в md-файле

## Обсуждение новых фич

Перед внедрением новой фичи создай документ в `docs/features/`:

```
docs/features/<feature-name>-proposal.md
```

### Обязательные разделы

- **Название фичи**
- **Описание проблемы**
- **Предлагаемое решение** (техническое)
- **Влияние на архитектуру** (слои FSD)
- **Альтернативы** и обоснование выбора
- **Оценка сложности** (время, риски)
- **Статус:** draft / approved / rejected / implemented

### Workflow

1. Proposal → обсуждение → approved
2. Реализация по FSD
3. Отчёт в `docs/reports/<feature-name>-report.md`
4. Коммит + тесты

---

## Тестирование

- **Каждая новая фича — с тестами.** Без тестов не мержить в main
- Минимальное покрытие: **80%** на модуль
- Тесты **детерминированные** и **независимые** друг от друга
- Моки для внешних сервисов (API, Google Drive, FS)

### Типы и расположение

| Тип | Инструмент | Где |
|-----|------------|-----|
| Unit | Jest | `__tests__/` или `*.test.ts` рядом с модулем |
| Integration | Jest + RTL | `features/<name>/__tests__/` |
| E2E | Detox / Maestro | `e2e/` |

### Стиль

- Один тест = один сценарий (Arrange → Act → Assert)
- Названия описывают **поведение**, не реализацию

```typescript
// ✅ should save note to local storage when offline
// ❌ should call AsyncStorage.setItem
```

### CI

- Тесты запускаются перед мержем; PR не мержится при падении
- Указывай в отчёте `docs/reports/` какие сценарии покрыты
