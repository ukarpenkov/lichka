# Анализ и план ускорения сборки Android

## Диагноз: почему сборка долгая

Нашёл **5 ключевых проблем** в `android/gradle.properties` и конфигах:

### 1. Parallel builds отключены (самое большое влияние)
```properties
# org.gradle.parallel=true   ← закомментировано!
```
Gradle собирает модули последовательно вместо параллельно. На multi-module проекте это критично.

### 2. Сборка под ВСЕ 4 архитектуры вместо одной
```properties
reactNativeArchitectures=armeabi-v7a,arm64-v8a,x86,x86_64
```
Для эмулятора нужна **только `x86_64`**. Сейчас компилируется 4x нативного кода (arm + x86).

### 3. Build cache отключён
Нет строки `org.gradle.caching=true` — каждый запуск пересобирает всё заново.

### 4. JVM heap маловат для New Architecture
```properties
org.gradle.jvmargs=-Xmx2048m
```
С `newArchEnabled=true` (codegen + Fabric + TurboModules) 2GB может не хватать — Gradle начит свопить.

### 5. Metro кэш не оптимизирован
`metro.config.js` — дефолтный, без кастомных настроек трансформера.

## План оптимизации

### Файл: `android/gradle.properties`

| Изменение | Было | Стало |
|-----------|------|-------|
| Parallel builds | `# org.gradle.parallel=true` | `org.gradle.parallel=true` |
| Build cache | (отсутствует) | `org.gradle.caching=true` |
| JVM heap | `-Xmx2048m` | `-Xmx4096m` |
| Архитектуры | `armeabi-v7a,arm64-v8a,x86,x86_64` | `x86_64` |

### Файл: `metro.config.js`

Добавить включённый файловый кэш Metro (по умолчанию он может быть выключен на некоторых платформах):
```js
const {getDefaultConfig, mergeConfig} = require('@react-native/metro-config');

const config = {
  transformer: {
    // Используем быстрый кэш трансформаций
    getTransformOptions: async () => ({
      transform: {
        experimentalImportSupport: false,
        inlineRequires: true,
      },
    }),
  },
};

module.exports = mergeConfig(getDefaultConfig(__dirname), config);
```

`inlineRequires: true` — выносит require()-вызовы из тела модуля, что ускоряет старт бандла.

### Скрипт: добавить npm-скрипт для быстрой сборки

В `package.json` добавить:
```json
"android:emulator": "react-native run-android --active-arch-only"
```
Флаг `--active-arch-only` заставляет Gradle собирать только архитектуру запущенного устройства/эмулятора.

## Ожидаемый эффект

| Оптимизация | Экономия |
|-------------|----------|
| Parallel builds | ~30-40% на повторных сборках |
| Одна архитектура (x86_64) | ~50-60% на нативной компиляции |
| Build cache | ~40-70% на повторных сборках |
| JVM 4GB | Убирает GC pauses и OOM |
| inlineRequires | Ускоряет Metro bundle |

**Итого**: чистая сборка ускорится в ~2-3 раза, повторная — в ~3-5 раз.

## Верификация

1. Выполнить `npm run android:emulator` на эмуляторе
2. Замерить время от запуска до появления "Running app on..."
3. Сравнить с предыдущим временем
